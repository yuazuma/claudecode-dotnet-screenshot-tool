# v1.1.0 動画生成機能 実装の思考記録

## 設計判断の記録

### APNG の実装方針

外部ライブラリ（ImageMagick.NET 等）を使わず、PNG チャンク仕様を直接実装した。
理由: SelfContained 単一バイナリを維持し、外部 DLL 依存を増やさないため。
CRC32 ルックアップテーブルを内包し、IHDR/acTL/fcTL/IDAT/fdAT/IEND を手書き。
最初のフレームは IDAT（標準 PNG 互換）、以降は fdAT（APNG 専用）。

### MediaFoundation COM Interop の課題

COM インターフェースのメソッド（`SetGUID`, `SetUINT32` 等）は `ref Guid` を要求するが、
リテラル `Guid` は `ref` に渡せない。ローカル変数に代入してから渡す必要がある。

解決策: `MfSetG(IMFMediaType t, Guid k, Guid v)` のようなヘルパーを作り、
内部でローカル変数を作って `ref` 渡しする。これにより呼び出し側はすっきり書ける。

H.264 エンコーダーへの入力は RGB32 bottom-up。`Bitmap.LockBits` で取得した
`Scan0` は top-down なので、行単位で逆順コピーが必要。

### WebP 対応

GDI+ は WebP を読めない。既に SixLabors.ImageSharp が依存グラフに入っているので、
それを流用して PNG メモリストリームに変換してから Bitmap を作る。
拡張メソッド `SaveAsPng()` は using 宣言なしには見つからないが、
`System.Drawing.Image` との名前衝突を避けるため、完全修飾の
`new SixLabors.ImageSharp.Formats.Png.PngEncoder()` を使う方針に切り替えた。

### TTS とフレーム時間の同期

Fixed モードでフレーム時間を固定秒にしているが、
TTS の読み上げ時間がそれより長い場合に動画と音声がずれる。
`Math.Max(fixedSec, wavDurationSec)` でフレームを延長することで対処。
WAV 時間の取得は純粋なヘッダパース（`System.Speech` の Duration プロパティは
MemoryStream からは読めないため、バイトオフセット計算で求める）。

### Notifier.ShowBalloon の欠落

VideoGenerator は Notifier を受け取り「動画生成を開始しました」「動画生成完了」を
バルーン通知する設計だった。しかし Notifier には汎用バルーンメソッドがなかった。
`ShowBalloonTip` のラッパーとして `ShowBalloon(title, message)` を追加した。
既存の `ShowManualGeneratedToast` 等と同じパターン（Dispatcher.BeginInvoke 経由）。

### 波紋アニメーションの実装

クリックイベントのフレームに 3 枚の波紋フレームを追加する。
半径 30→55→80px、アルファ 200→130→60 で外側に広がるリングを描画。
各波紋フレームは 0.15 秒、メインフレームは `dur - 0.15*3` 秒（最低 0.5 秒）。
APNG と MP4 で同じロジックを使う。

### SettingsWindow の条件表示パネル

フレームタイミング（固定/実時間）の選択によって、
「固定秒数」の入力欄と「最小/最大秒数」の入力欄を切り替える。
WPF の `Visibility` バインディングよりも、コードビハインドで
`UpdateFrameTimingPanels()` を呼ぶシンプルな方法を採用。
