## F-05-11: ファイル名トークン設計

**選択肢**:
- A) `ToString().ToLowerInvariant()` のまま（`mouseleftclick`）
- B) switch 式でマッピング（採用）
- C) `Enum.GetCustomAttribute` でカスタム属性

**B を選んだ理由**: switch 式は exhaustive（網羅性）が保証され、新しい `TriggerType` が追加された時にコンパイラ警告が出る。カスタム属性はシンプルなマッピングには過剰設計。

**トークン名の決定**: 要件定義書に「click」「drag」「scroll」「windowchange」「diff」が明示されていた。右クリック・中クリックは要件に名前指定がなかったため `rightclick` / `middleclick` と命名（直感的）。

## F-05-21: ディスク低容量→一時停止の設計

**アーキテクチャの課題**: `FileStorage` は `TriggerOrchestrator` を知らず、`NotifyIconWrapper` も知らない。疎結合を維持しながらどう「通知」するか。

**選択肢**:
- A) `FileStorage` に `TriggerOrchestrator` を DI する
- B) `FileStorage` に `Action?` コールバックを追加（採用）
- C) イベント（EventHandler）パターン

**B を選んだ理由**: `Action?` コールバックは最もシンプルで依存が少ない。`NotifyIconWrapper` がラムダで `_orchestrator.SetPaused(true)` と UI 更新をまとめてセットできる。すでに `SetNotifier()` パターン（メソッド注入）を使っているため一貫性がある。

**Dispatcher の必要性**: `CheckDiskSpace()` は `Task.Run` の `FireCapture()` 内から呼ばれるため、コールバックが UI スレッドで実行される保証がない。`OnLowDiskSpace()` 内で `Dispatcher.BeginInvoke` を使うことで確実にUIスレッドで一時停止操作を行う。

**二重一時停止の防止**: `if (_paused) return;` で既に一時停止中の場合は何もしない。

## F-06-08/09: オーバーレイ描画の設計

**座標変換の重要性**: スクリーンショットはモニター単位で撮影するため、カーソル座標（スクリーン絶対座標）をビットマップローカル座標に変換する必要がある:
```csharp
int localX = cursor.X - monitorBounds.X;
int localY = cursor.Y - monitorBounds.Y;
```
カーソルが別モニターにある場合は `localX < 0 || localX >= bmp.Width` となり早期リターン。

**描画順序**: 半透明塗りつぶし → 枠線 → クロスヘア。GDI+ は後から描いたものが上に来るため、クロスヘアが最前面になる。

**`SmoothingMode.AntiAlias`**: 円のギザギザを防ぐ。スクリーンショットは高解像度なのでアンチエイリアスが有効。

**半透明度の選択**: `Color.FromArgb(70, markerColor)` — アルファ値 70/255 ≒ 27%。背景が透けて見えつつマーカーも視認できる。100 以上だと背景が見えにくく、50 以下だとマーカーが薄すぎる。

## F-06-14: EventId の実装方式

**選択肢**:
- A) `TriggerEvent` のコンストラクタ引数に `Guid eventId` を追加
- B) 非位置引数プロパティ `Guid EventId { get; init; } = Guid.NewGuid()` を追加（採用）

**B を選んだ理由**: `TriggerOrchestrator.FireCapture()` の `new TriggerEvent(...)` 呼び出しを変更せずに済む。`init` で不変性を保ちつつ、レコードの `with` 式でコピー変更も可能。`Guid.NewGuid()` はインスタンス生成時に自動実行されるため、必ず一意な ID が付与される。

**フォーマット選択**: `ToString("N")` → ハイフンなし32文字（`5275aea9367c45f58f621b7617063f97`）。JSON の値として扱いやすく、ファイル名にも使える。

## F-08-05: キャプチャ履歴の設計

**表示タイミング**: `menu.Opening` イベントで動的構築する方式を選択。`BuildContextMenu()` 時点ではまだキャプチャがないため静的構築は不適切。Opening で毎回再生成することで常に最新の履歴を表示できる。

**件数制限**: `Queue<string>` で最大10件。`Queue` は FIFO なので古いものが先にDequeueされ、常に最新10件が保持される。`_recentPaths.Reverse()` で最新順に表示。

**ロック必要性**: `SaveAsync()` は `Task.Run` バックグラウンドスレッドから呼ばれ、`menu.Opening` は UI スレッドで呼ばれる。`lock (_recentPaths)` で両者を保護。
