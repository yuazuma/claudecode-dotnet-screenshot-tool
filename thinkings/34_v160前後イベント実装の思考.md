# v1.6.0 操作前後スクリーンショット実装の思考

## 設計上の主要な判断

### before/after の相関方法

**問題**: before は DOWN イベントで非同期に撮影される。after は UP + 遅延後に撮影される。
両者を同一ステップに紐付けるには相関が必要。

**採用案**: `_pendingBeforeShots[TriggerType]` にパスを一時保持し、
`FireCapture` 開始時に `TakeBeforeShot` で取り出す。

- 利点: シンプル、スレッドセーフ（lock で保護）、失敗時の degradation が容易
- 注意点: `MouseDragDrop` は `MouseLeftClick` と same key に正規化（DOWN 時は drag か click かわからない）
- 5 秒 expiry で孤立エントリを防止

**不採用案**: before/after を別ステップとして記録してから結合
→ データモデルが複雑になりすぎる

### HookService の遅延実装

`PostClickDelayMs` を Task.Delay で実装。Timer を使わない理由:
- Timer は GC 対策のためフィールドに保持が必要
- Task.Delay は自己完結型で扱いやすい
- ホットパスではないため Task 生成のオーバーヘッドは許容範囲

### 後方互換 JSON シム

`ProjectStep.LegacyImagePath` の `init` セッター:
- System.Text.Json は `init` セッターを .NET 7+ でサポート
- `get => null` + `WhenWritingNull` = 書き込み時は一切出力されない
- `init => AfterImagePath ??= value` = 読み込み時のみ適用

`??=` の重要性: もし JSON に `afterImagePath` と `imagePath` の両方があった場合（ありえないが）、
先に `afterImagePath` が設定された後、`imagePath` の `??=` は no-op になる。正しい動作。

### before 画像は PNG 固定

`FireBeforeCapture` で `_capture.Encode(bmp, Models.ImageFormat.Png)` を直接呼ぶ。
グローバル設定（`cfg.Storage.ImageFormat`）を無視することで、
ユーザーが JPEG に設定していても before は常にロスレス PNG になる。

### keyboard before のタイミング

`HookService._inKeyboardSession`:
- DOWN 時、フラグが false なら true に設定して `KeyboardBeforeEvent` 発火
- `TakeAccumulatedKeys()` でリセット（= after 撮影完了後に次のシーケンスが始められる）
- この設計でアイドル後の最初のキーだけが before を発火する

### ExportService での注意点

`ExportImagesAsync` での before エクスポートは、プロジェクトの `images/before/` から
直接コピーする（`AnnotationRenderer` を通さない）。before は証跡なのでアノテーションなし。

### ProjectViewWindow の XAML 変更

既存の `Grid.Row="0"` (高さ 260) を廃止し:
- Row 0: before 画像（`Auto`, 最大 120px, `Collapsed` when null）
- Row 1: アノテーション Canvas（230px 固定）
- Row 2: アノテーションツール（`Auto`）
- Row 3: ステップ詳細（`*`）

Window 全体の高さは 660 のまま。Row 0 が非表示の時は Row 1 が広く見える。
