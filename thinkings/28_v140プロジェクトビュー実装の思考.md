# v1.4.0 プロジェクトビュー UI リファクタリングの思考

## TabControl 廃止の判断

旧設計は「ステップ一覧」「ステップ詳細」「エクスポート」の3タブだったが、タブ切り替えが作業フロー上の摩擦になる。
特に「ステップ詳細を見ながらエクスポート」や「ステップを選びながらアノテーション」という操作は、タブをまたぐ必要があり非効率だった。
→ 縦分割レイアウトに切り替え、ステップ一覧と詳細を常時同時表示にした。

## 画像プレビュー高さの固定 vs 可変

`RowDefinition Height="260"` の固定にした。可変（*）にするとウィンドウリサイズ時に画像エリアが縦に引き伸ばされ、アノテーション座標変換のオフセット計算が毎回変わりUIが安定しない。
260px 固定なら `GetAnnTransform()` の結果も安定する。

## エクスポートドロップダウンの実装方針

WPF の `ContextMenu` は右クリックトリガーが標準。DropDownButton パターン（左クリックで展開）は ContextMenu を PlacementMode.Bottom で手動 IsOpen=true するのが最も単純。
`SplitButton` コントロールは標準 WPF には存在しないため、Button + ContextMenu の組み合わせを採用。

## ChkShowDeleted とインデックスの整合性

`_selectedProject.Steps` は全ステップのリスト、`_stepVms` はフィルタ後のリスト。
旧コードは `Steps[_selectedStepIndex]` を使っており、削除済み非表示時にインデックスがずれてしまう潜在バグがあった。
→ `_stepVms[i].Step` を参照するよう統一。`LoadSteps` でフィルタを適用し `_stepVms` を再構築する設計なので、インデックスは常に `_stepVms` を基準にするのが正しい。

## Expander の自動展開

アノテーションがない場合は Expander を折りたたむことで画面の縦スペースをステップ情報に割り当てる。
アノテーションが既にある場合は、編集中と判断できるため自動展開する。
`LoadAnnotationImage` 内で `_pendingAnnotations` に既存を追加した直後に判定するので、新規ステップは常に折りたたみ。

## SetStatus の busy フラグ

エクスポート中のフィードバックを文字だけでなく ProgressBar でも示す。
`IsIndeterminate=true` の ProgressBar は処理中を示すのに適切で、進捗率の計算コストが不要。
Collapsed/Visible の切り替えのみなので既存コードへの影響が最小。
