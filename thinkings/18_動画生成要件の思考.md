## ヒアリング設計の方針

### 質問の優先度設定

動画生成機能は出力フォーマット・視覚効果・音声・生成タイミング・技術選定の5軸がある。
アーキテクチャに影響する決定を先に聞き、詳細（テロップのフォントサイズ等）は後回しとする。

**第1バッチ（アーキテクチャ決定）**:
1. 出力フォーマット（MP4 / APNG / GIF の選択）
2. 動画の構成単位（セッション / チャプター）
3. フレーム表示時間（固定 / 実時間再現）
4. 音声（TTS / なし）

**第2バッチ（視覚・コンテンツ）**:
1. 強調表示スタイル（波紋 / 矩形枠 / カーソル / ズーム）
2. テロップ内容
3. TTS エンジン
4. 生成タイミング

**第3バッチ（品質・技術選定）**:
1. 出力解像度
2. 出力フォルダ
3. MP4 エンコード実装

## ヒアリング結果の分析

### MP4 エンコード: MediaFoundation を選択

ユーザーは MediaFoundation（Windows 標準 API）を選択した。
これにより:
- ffmpeg.exe の同梱が不要（100 MB 以上の削減）
- Windows 10 1809+ に標準搭載の H.264 MFT を使用
- `IMFSinkWriter` で MP4 コンテナに H.264 映像 + AAC 音声を多重化

代替案の評価:
- **FFmpegCore**: 高品質だが ffmpeg.exe 同梱が必須（配布サイズ増大）
- **Accord.Video**: メンテが古く .NET 8 との相性が不確か
- **MediaFoundation**: OS 組み込みのため追加バイナリ不要、要件に最適

### APNG: 純マネージド実装

APNG の生成ライブラリは NuGet に有力なものが少ない。
PNG チャンク構造（IHDR, fdAT, fcTL, IDAT, IEND）を直接書き込む実装を選択。

APNG の構造:
```
PNG シグネチャ
IHDR（解像度・色深度）
acTL（アニメーション制御: フレーム数・ループ回数）
[フレームごとに]
  fcTL（フレーム制御: 表示時間）
  fdAT（フレームデータ）
IEND
```

### TTS: Windows SAPI (System.Speech)

.NET 8 では `System.Speech.Synthesis.SpeechSynthesizer` は
`System.Speech` NuGet パッケージとして提供される（`net8.0-windows` TFM が必要）。

設計上の注意点:
- SpeechSynthesizer はシングルスレッド動作が推奨されるため、専用スレッドで実行
- `SpeakAsync` より `Speak` を別スレッドで実行する方が制御しやすい
- WAV ファイルへの出力: `SetOutputToWaveFile(path)` を使用

### 波紋アニメーション（フレーム展開）

動画はフレーム（静止画）の連続であるため、波紋を複数の中間フレームに展開する。
1クリックイベントにつき 3〜5 枚の波紋フレームを生成:

```
Frame 0: 元画像（波紋なし）
Frame 1: 半径 20px の薄い円
Frame 2: 半径 40px の中程度の円
Frame 3: 半径 60px のほぼ透明な円
```

GDI+ の `Graphics.DrawEllipse` + `Color.FromArgb(alpha, color)` でアルファを段階的に下げる。

## 技術的懸念点

### MediaFoundation の P/Invoke 複雑さ

MediaFoundation は COM ベースの API であり、P/Invoke による呼び出しは複雑。
`IMFSinkWriter`, `IMFMediaType`, `IMFSample`, `IMFMediaBuffer` 等の多数の COM インターフェースが必要。

SharpDX.MediaFoundation（`SharpDX.Direct3D11` + `SharpDX.MediaFoundation`）を使えば
.NET フレンドリーなラッパーが使えるが、SharpDX はメンテが停止している（2019年〜）。

→ 純粋な P/Invoke で実装するか、MediaFoundation の .NET 公式 Wrapper がある場合を調査する。
  Windows Community Toolkit の MediaFoundation 部分も検討対象。

### APNG と MP4 の同期

TTS 音声の長さとフレーム表示時間の同期が最も複雑な部分:
- APNG: 音声なし。フレーム表示時間は固定またはイベント間隔のみ
- MP4: TTS 読み上げ完了まで当該フレームを表示延長（固定間隔モード時）

`VideoGenerator` が `TtsService.SynthesizeAsync(text)` を呼び、
返却された WAV の長さ（秒）とフレーム設定値の max を取る。

### WebP 画像のデコード

撮影画像が WebP 形式の場合、GDI+ は WebP を直接読み込めない。
既存の `SixLabors.ImageSharp` を使って PNG/JPEG バイト列に変換してから
`System.Drawing.Image.FromStream()` に渡す。
