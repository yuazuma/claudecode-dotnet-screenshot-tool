## 配布パッケージの構成検討

### PublishSingleFile の扱い

`PublishSingleFile=true` + `IncludeNativeLibrariesForSelfExtract=true` を設定すると、
WPF ネイティブ DLL（wpfgfx_cor3.dll 等）が exe に内包される。

- exe サイズ: 約 180 MB（内包前）→ 約 188 MB（内包後）
- 利点: ユーザーに展開される DLL 本数が大幅に減る

一方、以下のマネージドアセンブリは single-file に含められず外部 DLL として残る:
- `Microsoft.Windows.SDK.NET.dll`（WinRT 投影層、22.5 MB）
- `DocumentFormat.OpenXml.dll`（6.1 MB）
- `SixLabors.ImageSharp.dll`（2.0 MB）
- Serilog 系 DLL
- `WinRT.Runtime.dll`

これら全部を exe と同じフォルダに置いて ZIP 化する。

### ZIP 圧縮の実装選択

`zip` コマンドがシステムにインストールされていなかったため、
Python の `zipfile.ZipFile` で対応した。

```python
with zipfile.ZipFile(zip_path, 'w', zipfile.ZIP_DEFLATED, compresslevel=5) as zf:
    for f in files:
        zf.write(src, arcname)
```

圧縮率: 220 MB (非圧縮) → 82 MB (圧縮後) = 約 63% 圧縮

`compresslevel=5` は速度と圧縮率のバランス。
9 にするとさらに小さくなるが処理時間が長くなる。5 で十分。

### .pdb ファイルの除外

デバッグシンボル（.pdb）は配布対象外。セキュリティ上もリバースエンジニアリングを
容易にするリスクがあるため、明示的に除外リストを作成してスキップした。

### SHA-256 チェックサム生成

```python
sha256 = hashlib.sha256()
with open(zip_path, 'rb') as f:
    for chunk in iter(lambda: f.read(65536), b''):
        sha256.update(chunk)
```

チャンクサイズ 65536 byte で大容量ファイルにも対応。
結果: `e008d0a9c84ff0b74cdc08b01a5a4b5ad757034ee06ce3499610b391dcd7e1ba`

### publish コマンドの注意点

実行前に `AutoScreenshot.exe`（PID 11188）が実行中だったため publish が失敗。
`taskkill //F //PID 11188` で強制終了してから再実行した。

```
dotnet publish -c Release -r win-x64 --self-contained true
  -p:PublishSingleFile=true
  -p:IncludeNativeLibrariesForSelfExtract=true
  -p:EnableCompressionInSingleFile=false
```

`EnableCompressionInSingleFile=false` は起動時の展開オーバーヘッドを避けるため。
true にすると exe サイズは小さくなるが起動が遅くなる。
