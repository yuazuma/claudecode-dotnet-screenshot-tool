## 作業方針の検討

### 読み込む対象ファイルの選定

21クラスをすべて実装レベルで記述するには、全ソースファイルを読む必要がある。
一方、ファイルが多いため「並列バッチ読み込み」で時間短縮する方針とした。

対象ディレクトリ構成:
```
src/
  App.xaml.cs
  Models/         (AppConfig, ManualGenConfig, ManualSession, TriggerEvent)
  Native/         (NativeMethods)
  Services/       (11サービスクラス)
  Views/          (SettingsWindow, ProgressWindow)
  Workers/        (TriggerOrchestrator, HookService)
```

以前のセッションで Explore agent が出力過多になって失敗したため、
今回は直接 Read ツールで重要クラスを1つずつ読む方針に変更。

### 設計書の粒度設定

「実装レベル」と「設計レベル」の中間点を選択した。

- 各メソッドのシグネチャ・引数・戻り値を記述
- アルゴリズムの要点（例: `CalcDiffRatio` の unsafe ピクセルループ）は擬似コードで記述
- 定数値・デフォルト値は実コードから転記
- WPF/XAML の詳細は省略（行動が主体のサービスクラスを重点化）

## 各クラスの実装解析

### CaptureService のオーバーレイ描画

`DrawImageOverlay` でトリガー種別ごとに色を変える。
- MouseLeft: 赤系 (255,70,0)
- MouseRight: 青系 (0,120,255)
- Keyboard: 緑系 (0,190,80)
- WindowChange: 紫系 (140,0,255)
- Diff: オレンジ系 (255,165,0)

左上に半透明の「●トリガー名」を描画する `BurnTimestamp` も同時適用。

### DiffDetector のパフォーマンス設計

`CalcDiffRatio` は unsafe コードで BitmapData に直接アクセスする。
`LockBits` → ピクセルポインタ走査 → `UnlockBits` のパターン。

差分判定基準:
- Manhattan距離 (|R1-R2| + |G1-G2| + |B1-B2|) > 30 のピクセルを「変化あり」とカウント
- 変化率が 30% を超えたらキャプチャイベントを発火

3秒インターバルで定期実行。Win32Exception (NativeErrorCode == 6) が出た場合は
「ウィンドウが最小化されている」とみなしてスキップ（エラーにしない）。

### HotkeyService の実装方針

WPF では `Window` を持たずに Win32 ホットキーを登録する必要があるため、
`NativeWindow`（WinForms 由来）を使って仮想的なウィンドウハンドルを作成。

```csharp
var hwnd = new NativeWindow();
hwnd.CreateHandle(new CreateParams { Style = 0 });
// HWND_MESSAGE (-3) は不可視のメッセージ専用ウィンドウ
```

`WM_HOTKEY (0x0312)` を受信したら `HotkeyPressed` イベントを発火。
`NotifyIconWrapper` がサブスクライブして一時停止/再開を切り替える。

### ManualSessionRecorder の依存関係

```
NotifyIconWrapper
  └─ ManualSessionRecorder
        ├─ UiaService
        ├─ OcrService
        └─ RuleBasedDescriber
```

`TriggerOrchestrator.CaptureCompleted` イベントを受け取って `RecordStepAsync` を呼ぶ。

### ConfigStore の永続化

`config.json` の場所優先順位:
1. exe と同じディレクトリ（ポータブルモード）
2. `%APPDATA%\AutoScreenshot\config.json`（既定）

`Update(Action<AppConfig>)` パターンにより、
呼び出し側が直接フィールドを変更→Save→ConfigChanged の一連を保証。

### DpapiHelper の設計

```csharp
ProtectedData.Protect(plainBytes, null, DataProtectionScope.CurrentUser)
```

`null` は追加エントロピーなし（シンプル化のため）。
`DataProtectionScope.CurrentUser` でログオンユーザーのみが復号可能。

暗号化データは Base64 文字列として `config.json` に保存。
`SettingsWindow` の LLMタブで入力→Protect、表示時は Unprotect してマスク表示（★×8固定）。

## 設計書の構成方針

```
1. システム概要（目的・アーキテクチャ）
2. 起動/終了フロー
3. 21クラスの詳細設計（クラスごとに責務・フィールド・主要メソッド）
4. データモデル（AppConfig / ManualGenConfig / ManualSession / TriggerEvent）
5. Win32 API一覧（P/Invoke宣言と用途）
6. 4つの処理フロー（キャプチャ/手順書生成/設定変更/ホットキー）
7. 設定・永続化
8. エラーハンドリング方針
9. セキュリティ設計
10. NuGetパッケージ一覧
```

全2232行。実装者が設計書だけで再実装できる詳細度を目標とした。
