# v1.5.0 プロジェクト機能有効無効除去の思考

## 除去の判断

`ProjectConfig.Enabled` は v1.1.0 互換のためだけに存在していた。
v1.4.0 時点でプロジェクト機能は全ユーザーに対して動作確認済みであり、
v1.1.0 動作に戻す理由はない。デッドコードを抱えたまま開発を続けることは
`if/else` 分岐の複雑度・テスト観点・見落としバグのリスクを増やすだけ。

## config.json 互換性

既存ユーザーの `config.json` に `"Enabled": false` が保存されている場合、
`ProjectConfig` にプロパティが存在しないため JSON デシリアライズ時に黙って無視される。
.NET の `System.Text.Json` / Newtonsoft.Json いずれも未知プロパティは既定でスキップするため
データ移行は不要。

## ManualGenConfig.OutputMarkdown / OutputDocx の扱い

手順書生成タブの「Markdown 出力」「Word 出力」チェックボックスはそのまま残した。
これらは `GenerateNow()` (手動生成) から `WriteSessionAsync` を経由して参照されていたが、
自動エクスポートのロジックは `ProjectConfig.AutoExportMarkdown/Docx` に一本化されたため、
`ManualGenConfig.OutputMarkdown/OutputDocx` は実質 dead config になる。
今回のスコープは「切り替えUI/ロジックの除去」なので、これらの dead config の整理は別タスクとする。

## VideoGen.AutoGenerateWithManual の扱い

同様に動画生成タブに残るが `WriteSessionAsync` からの参照がなくなった。
プロジェクト設定の `AutoExportVideo` が唯一の動画自動生成トリガーになる。

## NotifyIconWrapper の else ブロック除去

else 側（旧 v1.1.0 互換メニュー）には `手順書セッション区切り` / `手順書を生成` /
`動画を生成（GenerateVideoNow）` という3つの操作があった。
これらは ProjectViewWindow 経由で引き続き操作可能なため UX として問題なし。
`GenerateVideoNow` は VideoGenerator を直接渡す旧 API で、
プロジェクトモード（ExportService 経由）では `ExportVideoAsync` に統一されている。
