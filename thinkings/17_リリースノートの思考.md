## 既知の制限事項の記載判断

### WebP と docx の非互換

Open XML SDK は WebP 埋め込みに非対応（OOXML 規格自体が未対応）。
PNG・JPEG は正常に埋め込まれる。
→ ユーザーが Word 手順書を使う場合は PNG/JPEG 設定を推奨するため、明記する。

### SixLabors.ImageSharp CVE

`3.1.7` に中程度の CVE が報告されている。
ただし本ツールの使用方法（内部エンコードのみ・外部 WebP ファイルを読まない）では
実際の攻撃可能性は極めて限定的。次バージョンで更新予定として記載。

隠蔽すると後でユーザーから信頼を失うリスクがあるため、正直に記載する方針。

### LLM は Azure AI Foundry のみ

Anthropic API への直接接続は設計上サポートしない（セキュリティ・コスト管理方針）。
一般ユーザーが「Claude が使えるなら Anthropic API でも使えるのでは」と思い込む可能性があるため
明示的に制限として記載する。

## SHA-256 の記載方法

```
e008d0a9c84ff0b74cdc08b01a5a4b5ad757034ee06ce3499610b391dcd7e1ba  AutoScreenshot-v1.0.0-win-x64.zip
```

PowerShell での検証コマンドも日本語版には追記（ユーザーが実際に検証できるよう）:
```powershell
(Get-FileHash AutoScreenshot-v1.0.0-win-x64.zip -Algorithm SHA256).Hash
```
