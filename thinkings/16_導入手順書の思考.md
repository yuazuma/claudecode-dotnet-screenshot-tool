## 対象読者の設定

「要件定義書・ソースコード・これまでのプロンプト応答をもとに」という指示から、
作成対象はエンドユーザー向け（開発者ではなく利用者）と判断した。

想定読者:
- Windows PC を使う業務ユーザー
- .NET や開発環境の知識は不要
- 操作手順書（マニュアル）を作成したい業務担当者

→ 技術用語は最小限にとどめ、GUI 操作の手順を具体的に記述する方針。

---

## 章立ての設計

### 優先度の高い章

1. **インストール**: ZIP展開→exe実行の最小手順
2. **基本操作**: トレイアイコンの使い方（「これだけ知れば使える」）
3. **設定（全8タブ）**: 各設定項目の意味と推奨値
4. **操作手順書の作成**: メインユースケースの完全手順

### 優先度の低い章（参照用）

5. 自動起動設定（オプション）
6. グローバルホットキー（オプション）
7. アンインストール（必要時のみ）
8. ファイル構成（%APPDATA% の説明）
9. トラブルシューティング（問題時の参照）

---

## 設定タブの説明方針

8タブ（基本/撮影/キーボード/除外/操作手順/LLM/テンプレート/ホットキー）の各設定項目は、
`SettingsWindow.xaml.cs` の `LoadSettings()` / `ApplySettings()` を参照して
設定名・型・デフォルト値を確認した。

各設定項目に対して:
- 何の設定か（1行説明）
- 変更が必要なケース（「〜の場合は変更してください」）
- 推奨値またはデフォルト値

という形式で記述。

### LLM（Azure AI Foundry）タブの記述

DPAPI 暗号化の存在はユーザーに意識させない。
「設定画面に API キーを入力すると暗号化して保存されます」程度の説明に留める。
「Windows DPAPI」「ProtectedData」などの技術用語は使わない。

## トラブルシューティングの項目選定

実装コードと過去のデバッグセッション（`documents/14_要件定義書動作確認.md` 等）から
実際に発生した問題を抽出:

1. 起動時の SmartScreen 警告 → 「詳細情報」→「実行」
2. タスクトレイにアイコンが表示されない → システムトレイの隠れたアイコンを確認
3. 撮影されない → 除外アプリ設定・一時停止状態を確認
4. 手順書が空になる → セッション開始後に操作したか確認
5. OCR が機能しない → Windows 言語パックの確認
6. 多重起動エラー → Mutex による単一インスタンス制御（すでに起動中）
7. 設定が保存されない → %APPDATA% への書き込み権限確認
