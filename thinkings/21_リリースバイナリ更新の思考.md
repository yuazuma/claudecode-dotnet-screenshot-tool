# v1.1.0 リリースバイナリ更新の思考記録

## 再ビルドが必要だった理由

前セッション（v1.1.0 実装）で `dotnet publish` を実行済みだったが、
現在の `publish/` ディレクトリに `System.Speech.dll` が存在しなかった。

考えられる原因:
1. 前セッションの publish は `System.Speech 9.0.0` パッケージ追加前のビルドだった
2. または publish 後にプロセス終了のために `taskkill` でアプリを停止したが、
   その際に publish ディレクトリをクリアする操作が入った可能性
3. または前回の publish 結果が別ディレクトリに出力されていた

`System.Speech.dll` なしで ZIP を作ると TTS 機能が実行時 `FileNotFoundException` になるため、
確認後に再ビルドを選択した。

## dotnet コマンドのパス解決

Bash ツールから直接 `dotnet` を呼ぶとコマンドが見つからない（`command not found`）。
これは Bash ツールのシェル環境に dotnet が PATH に入っていないため。

解決策: フルパス `"C:\Program Files\dotnet\dotnet.exe"` を指定。
（`powershell -Command` 経由は deny ルールに抵触するため使用しない）

## ZIP 作成の実装選択

Bash の zip コマンドや 7-Zip ではなく Python の `zipfile` モジュールを使用した。
理由:
- Python は環境に確実に存在する（Bash ツールで確認済み）
- `zipfile.ZIP_DEFLATED` + `compresslevel=6` で標準的な圧縮
- `hashlib.sha256()` も同じスクリプト内でまとめて処理できる

## v1.1.0 ZIP が v1.0.0 より小さい理由

v1.0.0 ZIP: 86,209,829 bytes（約 82 MB）
v1.1.0 ZIP: 81,757,399 bytes（約 78 MB）

差分: -4.5 MB

v1.1.0 で追加した `System.Speech.dll` は既に .NET ランタイムに内包されているため、
self-contained publish では .NET SDK のバージョンアップ（8.0.x）による
ランタイム DLL の変更（特に圧縮効率の改善や削除）が影響している可能性がある。
`System.Speech.dll` 自体の追加（〜1MB 程度）よりも、
.NET 8.0.x ランタイムの変化の方が大きく効いていると考えられる。

## v1.0.0 ZIP の保持

v1.0.0 の ZIP とハッシュを削除しなかった。
理由: 過去バージョンへのダウングレード需要や、
配布履歴の証跡として残すのが releases/ ディレクトリの一般的な慣行。
また、ユーザーが「v1.0.0に戻したい」場合の参照ファイルとしても機能する。

## ハッシュのリリースノート反映

SHA-256 ハッシュをリリースノートに埋め込んだことで、
配布ファイルとリリースノートのどちらを見ても整合性確認が可能になる。
`.sha256` ファイルは `{ハッシュ}  {ファイル名}` という形式（shasum コマンド互換）で
`shasum -c` / `certutil -hashfile` で検証可能。
