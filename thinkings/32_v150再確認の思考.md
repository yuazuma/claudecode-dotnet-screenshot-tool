# v1.5.0 再確認セッション（RDP-Tcp#1）の思考

## 背景

前セッション（RDP-Tcp#0, PID 10924）でトレイアイコン周辺の画面キャプチャに問題が
発生し、クリーンビルド後の再起動 exe（PID 10876, RDP-Tcp#1）でのトレイ操作が
未完了のまま context が圧縮された。本セッションでその続きを実施した。

## UIPI ブロックの問題

### 症状

```python
user32.SetCursorPos(1470, 1043)   # returns 0 (failure)
user32.SendInput(...)              # returns 0 (events injected = 0)
ctypes.get_last_error()           # returns 0 (no error code)
```

`SetCursorPos`・`SendInput` ともにエラーコード 0 で失敗。
`GetCursorPos` が常に (0, 0) を返す。

### 原因

Windows Vista 以降の UIPI（User Interface Privilege Isolation）による制限。
- `SendInput` が 0 を返し、`GetLastError()` が 0 の場合 →「入力が別スレッドにブロックされた」
- Claude Code VS Code 拡張から起動された Python サブプロセスは、
  インタラクティブデスクトップへの入力インジェクション権限を持たない
- デスクトップは `WinSta0\Default` に正しく所属しているが、
  フォアグラウンドプロセスでないため入力注入が拒否される

### 影響

- `mouse_event`、`SetCursorPos`、`SendInput` すべて不可
- トレイアイコンへの直接クリックによる UI 確認が不可能

## 代替アプローチ：バイナリ文字列検索

マウス操作なしで以下を検証できる:

1. **PE バージョンリソース**（EXE）: `GetFileVersionInfoW` で FileVersion を確認
2. **DLL バイナリ文字列検索**: .NET マネージドアセンブリに埋め込まれた文字列リテラルを
   `data.find(s.encode('utf-16-le'))` で検索

### 検索結果（AutoScreenshot.dll, 294,400 bytes）

| 検索対象 | エンコード | 結果 |
|---|---|---|
| `ChkProjectEnabled` | UTF-16LE | NOT FOUND ✅ |
| `ChkProjectEnabled` | UTF-8 | NOT FOUND ✅ |
| `projectEnabled` | UTF-8 | NOT FOUND ✅ |
| `AutoScreenshot v1.5.0` | UTF-16LE | FOUND（offset 230012）✅ |
| `1.5.0.0` | UTF-16LE | FOUND（offset 209426）✅ |

### なぜこれが有効な証拠か

- `ChkProjectEnabled` は XAML のコントロール名（`x:Name`）
  → BAML リソース（XAML を IL にコンパイルしたもの）に埋め込まれる
  → 削除されていれば DLL に文字列として存在しない
- `projectEnabled` は `BuildContextMenu()` の local 変数名
  → C# コンパイラが生成する debug metadata（PDB は別ファイルだが）や
    MVVM binding path として残ることがある
  → いずれも NOT FOUND → コードから完全除去されている
- `AutoScreenshot v1.5.0` は `NotifyIconWrapper.BuildVersionItem()` 内のリテラル
  → コンパイル済みアセンブリに UTF-16LE 文字列として埋め込まれる
  → FOUND で v1.5.0 への更新を確認

## Shell_NotifyIconGetRect による座標確認

```python
# UID=1 で成功（S_OK）
# rect: (2181, 1528)-(2229, 1600) （物理座標）
```

- 物理座標中心: (2205, 1564) → 論理座標 (1470, 1043) (DPI 1.5x)
- アイコンが UID=1 で登録されていることを確認（WinForms NotifyIcon のデフォルト UID）
- UIPI により実際のクリックには使用できなかったが、アイコンの存在は確認

## 結論

前セッションの PASS 判定（スクリーンショット付き）に加え、
クリーンビルド後の DLL バイナリ検証でも同一の結論が得られた。
v1.5.0 の要件（`ProjectConfig.Enabled` 切り替え除去）はバイナリレベルで確認済み。
