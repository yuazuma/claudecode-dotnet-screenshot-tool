## 1. システム概要

### 1.1 目的

AutoScreenshot は Windows デスクトップ上の操作を自動記録するシステムトレイ常駐型アプリケーションである。マウス・キーボード・ウィンドウ切替などのトリガーを検知してスクリーンショットを自動撮影し、撮影済みの操作ログから業務操作手順書（Markdown / Word docx）を自動生成する。

### 1.2 主な機能領域

| 機能領域 | 概要 |
|---|---|
| スクリーンショット撮影 | トリガー検知→全モニタ一括撮影→保存 |
| 操作イベントロギング | テキスト/JSONL/CSV形式でのメタデータ記録 |
| 手順書自動生成 | セッション単位でMarkdown/Word(.docx)を出力 |
| UI要素情報取得 | UIA（Windows UI Automation）/ OCRによる操作対象コントロール特定 |
| LLM連携 | Azure AI Foundry（Claude）による説明文改善・ダイジェスト生成 |
| **動画自動生成** | **セッション単位でAPNG / MP4動画を出力（TTS音声ナレーション付き）** |
| **HTML エクスポート** | **単一 HTML ファイル（Base64 画像埋め込み）による自己完結型手順書出力（FR-A）** |
| **インクリメンタル LLM** | **ステップ記録ごとにバックグラウンドで LLM 説明文改善をキューイング（FR-B）** |
| **ステップアノテーション** | **番号バッジ/矢印/矩形/吹き出しを画像に焼き込みエクスポート（FR-C）** |
| **プロジェクト管理強化** | **全文検索・タグフィルタ・ドラッグ&ドロップ並び替え・手動ステップ挿入（FR-D）** |
| **プロジェクト結合・分割** | **複数プロジェクト結合／指定ステップでの分割（元プロジェクト保持）（FR-E）** |
| 設定管理 | JSONベース設定・DPAPI暗号化・ポータブル運用対応 |

### 1.3 技術スタック

| 分類 | 採用技術 |
|---|---|
| 言語 | C# 12 / .NET 8.0 |
| UIフレームワーク | WPF（設定ウィンドウ・ダイアログ）+ WinForms（トレイアイコン） |
| キャプチャ | GDI+（System.Drawing）|
| 画像エンコード | PNG/JPEG: .NET 標準 / WebP: SixLabors.ImageSharp |
| Word生成 | DocumentFormat.OpenXml 3.1.0 |
| LLM | Azure.AI.Inference 1.0.0-beta.5（`ChatCompletionsClient`）|
| 暗号化 | Windows DPAPI（System.Security.Cryptography.ProtectedData）|
| ロギング | Serilog（日次ローリングファイル）|
| Win32連携 | P/Invoke（グローバルフック・WinEventフック・ホットキー等）|
| 動画生成 | APNG: 純マネージドPNGチャンク書き込み / MP4: Windows MediaFoundation COM P/Invoke（IMFSinkWriter）|
| TTS | Windows SAPI（System.Speech.Synthesis.SpeechSynthesizer）|

## 4. クラス設計

### 4.1 HookService

| 項目 | 内容 |
|---|---|
| 名前空間 | AutoScreenshot.Services |
| 依存 | `Func<TriggerConfig>`（設定参照） |
| 公開イベント | `MouseEvent(TriggerType)`, `KeyboardActivity`, `ActiveWindowChanged` |

**公開メソッド:**

| メソッド | 説明 |
|---|---|
| `Start()` | グローバルフック3種（マウス/キーボード/WinEvent）を設置 |
| `Stop()` | フックを解除・タイマー停止 |
| `TakeAccumulatedKeys()` | `(string inputText, string keyCodes)` を返してバッファをクリア |

**内部状態:**

| フィールド | 型 | 説明 |
|---|---|---|
| `_inputTextBuf` | StringBuilder | 印刷可能文字の蓄積 |
| `_keyCodeBuf` | List\<string\> | `Ctrl+C` 等キー名の蓄積 |
| `_shiftDown/_ctrlDown/_altDown` | bool | 修飾キー状態 |
| `_lbDownTime` | DateTime | ドラッグ検知用左ボタン押下時刻 |

**キーボードバッファロジック（E-04実装）:**

```
KeyboardHookCallback(WM_KEYDOWN)
  → vkCode をKBDLLHOOKSTRUCTから読取
  → 修飾キー状態を更新
  → AccumulateKey(vk)
       ├─ Ctrl/Shift/Alt単体 → 無視
       ├─ Backspace → _inputTextBuf の末尾1文字削除
       ├─ 印刷可能文字 → _inputTextBuf に Append
       └─ 全キー → _keyCodeBuf に "Ctrl+A" 形式で Add
```

### 4.2 TriggerOrchestrator

| 項目 | 内容 |
|---|---|
| 依存 | ConfigStore, HookService, CaptureService, FileStorage, DiffDetector, MetadataLogger, Notifier, MaskingService, ManualSessionRecorder（任意） |

**FireCapture シグネチャ:**

```csharp
private void FireCapture(
    TriggerType trigger,
    IReadOnlyList<int>? screenIndices = null,
    (string inputText, string keyCodes)? keyboardInput = null)
```

**クールダウン管理:** `_lastCapture[TriggerType]` で各トリガーの最終撮影時刻を管理。指定秒数内の再トリガーは破棄。

**差分検知タイマー:** 設定値（既定3秒）のインターバルでサムネイル比較。直前1秒以内にマウス/キーボード操作があった場合は除外（操作起因の差分を重複記録しない）。

**除外アプリ判定:** `PrivacyConfig.ExcludeApps` のリストに対してワイルドカード（`*`）マッチング。プロセス名・ウィンドウタイトルの両方をチェック。

### 4.3 ManualSessionRecorder

| 項目 | 内容 |
|---|---|
| 依存 | ConfigStore, UiaService, OcrService, Notifier（任意） |
| スレッド安全 | `_lock` による `_current` の排他制御 |

**メソッド一覧:**

| メソッド | 同期/非同期 | 説明 |
|---|---|---|
| `StartSession(title)` | 同期 | ManualSession 生成・セッション開始ログ |
| `RecordStepAsync(evt, path, bounds)` | 非同期 | UIA/OCR→説明文生成→ManualStep追加 |
| `SplitSession(nextTitle)` | 非同期（Task.Run） | 現セッション保存→新セッション開始（S-03） |
| `GenerateNow()` | 非同期（Task.Run） | 現ステップのスナップショットで手順書生成 |
| `StopSessionAsync()` | 非同期 | セッション終了・手順書保存（S-02） |
| `SetVideoGenerator(gen)` | 同期 | VideoGeneratorインスタンスを注入（NotifyIconWrapperが呼び出す） |
| `GenerateVideoNow(gen)` | 非同期（Task.Run） | 現セッションのスナップショットで動画を即時生成 |

**RecordStepAsync フロー:**

```
UIA.GetElementAt/GetFocused() [lock外、Task.Run、200msタイムアウト]
  ├─ 成功 → (uiName, ctrlType)
  └─ 失敗 → OcrService.RecognizeNearbyText() [±40px切出し]
               ├─ 成功 → uiName
               └─ 失敗 → needsReview = true

KeyboardMode判定（E-04）:
  Both      → InputText=evt.InputText, KeyCodes=evt.KeyCodes
  RealText  → InputText=evt.InputText, KeyCodes=null
  KeyCode   → InputText=null, KeyCodes=evt.KeyCodes

ManualStep 構築 → DescriptionRuleBased 生成 → _current.Steps.Add()
```

**WriteSessionAsync フロー:**

```
ステップ0件 → 早期リターン（ログのみ）
LlmEnabled && endpoint && apiKey 設定済み:
  ImproveDescriptionsAsync() → DescriptionLlm 更新
  GenerateDigestAsync()      → session.Digest 設定
  失敗 → Warning ログ → ルールベースで継続
OutputMarkdown → MarkdownManualWriter.WriteAsync()
OutputDocx     → DocxManualWriter.WriteAsync()
Notifier.ShowManualGeneratedToast(llmUsed)
```

### 4.4 MarkdownManualWriter

**出力ファイル構成（GFM準拠）:**

```markdown

- **開始日時**: ...
- **終了日時**: ...
- **OS / ユーザー**: ...
```

- `WindowTitle` が空でない → そのまま使用

- 両方空 → `(不明なウィンドウ)` を使用

**テンプレート適用（O-07）:**
1. テンプレートファイルを読み込む
2. `{{content}}` プレースホルダーが存在 → 置換
3. プレースホルダーなし → テンプレート末尾に生成コンテンツを追記

### 4.5 DocxManualWriter

**Open XML構成:**

| 要素 | 実装 |
|---|---|
| スタイル | Normal, Title, Heading1/2/3（カスタム定義） |
| ヘッダー | 右寄せでセッションタイトル |
| フッター | 中央にページ番号フィールド（`<w:fldChar>/<w:instrText>PAGE</w:instrText>`）|
| 表紙 | Titleスタイル段落 + 改ページ |
| 目次 | TOCフィールド（`TOC \o "1-3" \h \z \u`）+ 改ページ |
| 画像 | InlinePicture、最大幅5400000EMU（約15cm）、縦横比維持 |

**テンプレート対応（O-07）:**
```
.dotx をコピー → WordprocessingDocument.Open() → ChangeDocumentType(docx)
→ Body.Elements<Paragraph>() を除去（SectionProperties保持）
→ 生成コンテンツを追加（テンプレートスタイルを継承）
```

テンプレート読み込み失敗時は `Log.Warning` → 通常作成（フォールバック）。

### 4.6 LlmService

**APIクライアント設定:**

```csharp
var client = new ChatCompletionsClient(
    new Uri(endpoint),
    new AzureKeyCredential(apiKey));
// モデル名はオプションに設定（Foundryデプロイメント名）
options.Model = _deploymentName;
```

**ImproveDescriptionsAsync（L-03）:**
- システムプロンプト: 操作手順書の説明文改善に特化した日本語プロンプト
- ユーザーコンテンツ: `{番号}. {DescriptionRuleBased}` を列挙
- レスポンスをパース: 正規表現 `^(\d+)\.\s+(.+)$` で番号→説明文を抽出
- 対応するステップの `DescriptionLlm` を更新

**GenerateDigestAsync（L-04）:**
- 全ステップのウィンドウタイトル・トリガー・UI名をテキスト化
- 画像データは送信しない（L-05）
- 3〜5行の操作要約を生成
- 戻り値 `string?`（失敗時null）

### 4.12 VideoGenerator

| 項目 | 内容 |
|---|---|
| 名前空間 | AutoScreenshot.Services |

### 4.13 FrameRenderer

**RenderFrame（フレーム合成）:**

### 4.14 ApngWriter

**APNG構成（PNG拡張仕様準拠）:**

```
PNG Signature (8 bytes)
IHDR  ← 幅/高さ/BitDepth=8/ColorType=2(RGB)
acTL  ← num_frames, num_plays=0（無限ループ）
fcTL[0] + IDAT  ← 第1フレーム（アニメーションのデフォルト画像を兼ねる）
fcTL[1] + fdAT  ← 第2フレーム以降
...
IEND
```

**フレームタイミング（VideoGenConfig.FrameTimingMode）:**

| モード | 動作 |
|---|---|
| Fixed | `FixedFrameSeconds` をすべてのフレームに適用 |
| Variable | TTS WAV の実際の時間長を fcTL の delay_num/delay_den に設定（最小 `MinFrameSeconds`、最大 `MaxFrameSeconds` でクリップ） |

```csharp
[DllImport("mf.dll")] MFStartup / MFShutdown

```

**ビデオストリーム設定:**

| 属性 | 値 |
|---|---|

| MF_MT_AVG_BITRATE | Mp4VideoBitrateMbps × 1,000,000 bps |
| 入力フォーマット | MFVideoFormat_RGB32（上下反転してBitmapからコピー） |

**オーディオストリーム設定（TTS有効時）:**

| 属性 | 値 |
|---|---|
| MF_MT_SUBTYPE | MFAudioFormat_AAC |
| MF_MT_AUDIO_SAMPLES_PER_SECOND | 44100 Hz |
| MF_MT_AUDIO_NUM_CHANNELS | 1 |
| MF_MT_AUDIO_AVG_BYTES_PER_SECOND | 12000 |
| 入力フォーマット | MFAudioFormat_PCM（WAVファイルのPCMデータを直接供給）|

## 7. 処理フロー

### 7.1 アプリ起動フロー

```
App.OnStartup()
  1. Mutex取得（"AutoScreenshot_SingleInstance"）
     → 取得失敗: MessageBox表示 → Shutdown()
  2. ConfigureLogging()
     → Serilog RollingFile: %APPDATA%/AutoScreenshot/logs/app-.log（日次）
  3. NotifyIconWrapper生成
       ConfigStore.Load()
       各サービスをnew
       TriggerOrchestrator生成（サービス群を注入）
       HookService.Start()
       ManualGen.Enabled → ManualSessionRecorder.StartSession(GetSessionTitle())
       HotkeyService.Register()
       AutoStartService.Ensure()
       NotifyIcon表示 + メニュー構築
  4. Log.Information("AutoScreenshot 起動")
```

### 7.2 トリガー検知→撮影フロー

```
[マウス系]
HookService.MouseHookCallback(WM_LBUTTONUP)
  → _lbDownTime からドラッグ判定
  → MouseEvent?.Invoke(MouseLeftClick or MouseDragDrop)
  → TriggerOrchestrator.OnMouseEvent()
       IsExcluded? → return
       CheckCooldown? → return
       Log.Debug("撮影キュー投入")
       FireCapture(trigger)

[キーボード系]
HookService.KeyboardHookCallback(WM_KEYDOWN)
  → 修飾キー状態更新
  → AccumulateKey(vk) → _inputTextBuf / _keyCodeBuf に蓄積
  → KeyboardActivity?.Invoke()
  → TriggerOrchestrator.OnKeyboardActivity()
       _keyboardIdleTimer.Dispose() → new Timer(idleSeconds後にFire)
       タイマー発火時:
         TakeAccumulatedKeys() → (inputText, keyCodes)
         FireCapture(Keyboard, keyboardInput:(inputText, keyCodes))

[ウィンドウ切替系]
HookService.WinEventCallback(EVENT_SYSTEM_FOREGROUND)
  → ActiveWindowChanged?.Invoke()
  → TriggerOrchestrator.OnActiveWindowChanged()
       CheckCooldown → FireCapture(ActiveWindowChange)

FireCapture(trigger, ...) [Task.Run内]
  GetCursorPos() / GetActiveWindowTitle() / GetActiveProcessName()
  CaptureService.CaptureAllScreens()
  foreach (bmp, monitorIdx, bounds):
    TriggerEvent生成（InputText/KeyCodes設定）
    MaskingService.ApplyMasking(bmp, bounds)   ← Privacy設定時
    CaptureService.DrawImageOverlay(...)        ← ImageOverlay設定時
    CaptureService.BurnTimestamp(...)           ← BurnTimestamp設定時
    byte[] data = CaptureService.Encode(bmp, format, quality)
    path = await FileStorage.SaveAsync(data, format, evt)
    await MetadataLogger.LogEventAsync(evt, path)
  Notifier.OnCaptured()
  await ManualSessionRecorder.RecordStepAsync(stepEvt, firstPath, firstBounds)
```

### 7.3 手順書セッション終了→生成フロー

```
[正常終了: トレイメニュー「終了」]
Application.Current.Shutdown()
  → App.OnExit()
  → NotifyIconWrapper.Dispose()
       HookService.Stop()
       ManualSessionRecorder.StopSessionAsync().GetAwaiter().GetResult()
         WriteSessionAsync(session)
           LLM処理（有効時）
           MarkdownManualWriter.WriteAsync() / DocxManualWriter.WriteAsync()
           Notifier.ShowManualGeneratedToast(llmUsed)
       NotifyIcon.Dispose()
  → Log.CloseAndFlush()

[セッション分割: 手順書セッション区切り]
ManualSessionRecorder.SplitSession(GetSessionTitle())
  Task.Run:
    completed = _current → WriteSessionAsync(completed)
    _current = null → StartSession(nextTitle)

[即時生成: 手順書を生成]
ManualSessionRecorder.GenerateNow()
  Task.Run:
    snapshot = shallow copy of _current
    WriteSessionAsync(snapshot)   ← _current は継続
```

### 7.4 動画生成フロー

```
[トレイメニュー「動画を生成」]
ManualSessionRecorder.GenerateVideoNow(_videoGenerator)
  Task.Run:
    snapshot = shallow copy of _current
    await _videoGenerator.GenerateAsync(snapshot)

[手順書生成と同時生成: AutoGenerateWithManual=true]
ManualSessionRecorder.WriteSessionAsync(session)
  MarkdownManualWriter / DocxManualWriter → 手順書生成
  VideoGenConfig.AutoGenerateWithManual → await _videoGenerator.GenerateAsync(session)

[VideoGenerator.GenerateAsync(session)]
  _semaphore.Wait() ← 多重実行を防止
  frames = session.Steps.Where(s => File.Exists(s.ImagePath)).ToList()
  frames が空 → Warning → return

  // TTSフェーズ
  wavPaths = new List<string?>()
  foreach frame in frames:
    if TtsEnabled && !string.IsNullOrEmpty(frame.Description):
      wav = await TtsService.GenerateWavAsync(frame.DescriptionRuleBased, cfg)
      wavPaths.Add(wav)
    else:
      wavPaths.Add(null)

  // フレーム合成フェーズ
  renderedBitmaps = new List<Bitmap>()
  foreach frame in frames:
    bmp = FrameRenderer.RenderFrame(frame, cfg)  ← 解像度変換 + 波紋/矩形/テロップ合成
    renderedBitmaps.Add(bmp)

  // フレーム表示時間の決定
  delays = CalcDelays(wavPaths, cfg)  ← Fixed or Variable

  // APNG出力
  if OutputApng:
    await ApngWriter.WriteAsync(outputPath, renderedBitmaps, delays)

  // MP4出力
  if OutputMp4:
    await MfVideoWriter.WriteAsync(outputPath, renderedBitmaps, delays, wavPaths, cfg)

  Notifier.ShowBalloon("動画生成完了", outputFolder)
  if OpenFolderOnComplete: Process.Start("explorer.exe", outputFolder)
  _semaphore.Release()
```

## 10. セキュリティ設計

### 10.1 API資格情報の保護（NF-04）

| 項目 | 方針 |
|---|---|
| 保存場所 | config.json（AppData/Roaming） |
| 暗号化 | Windows DPAPI、CurrentUserスコープ |
| 転送 | TLS（HTTPS）でFoundryエンドポイントと通信 |
| 表示 | SettingsWindow でPasswordBox（非echo）使用 |
| LLMへの画像送信 | 禁止（L-05）、テキスト情報のみ送信 |

### 10.2 プライバシー保護

| 機能 | 実装 |
|---|---|
| パスワード欄マスキング | UIAutomation `IsPasswordProperty` で検出、GDI+で黒塗り |
| アプリ除外 | プロセス名/ウィンドウタイトルのワイルドカードマッチング |
| ローカル保存 | スクリーンショットはローカルファイルシステムのみ |

### 10.3 二重起動防止

Named Mutex `AutoScreenshot_SingleInstance` でシングルインスタンスを保証。起動試行時に既存インスタンスが検知された場合はメッセージ表示後に終了。

## 11. 非機能要件への対応

| ID | 要件 | 実装方針 |
|---|---|---|
| NF-01 | 手順書生成のUIブロック禁止 | `SplitSession()`/`GenerateNow()` は `Task.Run` で非同期実行 |
| NF-02 | 100ステップLLMなし3秒以内 | Markdown生成はString操作のみ、File I/Oは`async/await` |
| NF-03 | LLM完了トースト通知 | `Notifier.ShowManualGeneratedToast(llmUsed)` — 手順書書出し後に呼び出し |
| NF-04 | APIキー・エンドポイントURL DPAPI暗号化 | `DpapiHelper.Protect/Unprotect` with CurrentUser scope |
| NF-05 | UIA 200msタイムアウト | `Task.WhenAny(task, Task.Delay(200))` |
| NF-06 | Windows.Media.Ocr優先 | `OcrEngine.TryCreateFromLanguage()` — 失敗時null、Tesseractフォールバックは未実装（将来対応） |
| NF-07 | DocumentFormat.OpenXml使用 | DocxManualWriter.cs で全面採用 |
| NF-V01 | 動画生成のUIブロック禁止 | `VideoGenerator.GenerateAsync()` は呼び出し側が `Task.Run` で実行 |
| NF-V02 | 動画の多重生成防止 | `SemaphoreSlim(1,1)` で同時実行数を1に制限。生成中の「動画を生成」メニュー呼び出しは即時リターン |
| NF-V03 | MediaFoundation初期化 | `MFStartup(MF_VERSION, MFSTARTUP_NOSOCKET)`  —  COM初期化は呼び出しスレッドが担う |
