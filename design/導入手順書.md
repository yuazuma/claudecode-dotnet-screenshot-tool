## 1. 動作環境

| 項目 | 要件 |
|---|---|
| OS | Windows 10 バージョン 1809 (October 2018 Update) 以降、または Windows 11 |
| アーキテクチャ | x64 (64 ビット) のみ |
| .NET ランタイム | **不要**（自己完結型バイナリに .NET 8 を内包） |
| ディスク空き容量 | インストール: 約 300 MB（スクリーンショット保存分は別途） |
| メモリ | 常駐時 約 100 MB（目安） |
| 権限 | **管理者権限不要**（標準ユーザー権限で動作） |
| ネットワーク | 基本機能は不要。LLM 連携を使う場合は Azure AI Foundry への HTTPS 接続が必要 |

> **注意**: 仮想マシン（VM）環境や RDP セッションでも動作しますが、ディスプレイが利用不可になる操作（スリープ、セッション切断等）の瞬間は差分検知が一時スキップされます。

## 2. インストール

### 2.1 配布 ZIP からのインストール

#### ステップ 1: ZIP を展開する

配布ファイル `AutoScreenshot-v1.1.0-win-x64.zip` を任意のフォルダに展開します。

```
C:\Tools\AutoScreenshot\          ← 展開先の例（任意）
  ├── AutoScreenshot.exe
  ├── AutoScreenshot.dll
  ├── AutoScreenshot.runtimeconfig.json
  ├── AutoScreenshot.deps.json
  ├── DocumentFormat.OpenXml.dll
  ├── DocumentFormat.OpenXml.Framework.dll
  ├── Microsoft.Windows.SDK.NET.dll
  ├── Serilog.dll
  ├── Serilog.Sinks.File.dll
  ├── Serilog.Sinks.Console.dll
  ├── SixLabors.ImageSharp.dll
  ├── System.Speech.dll
  ├── WinRT.Runtime.dll
  └── README.txt
```

> **重要**: `AutoScreenshot.exe` と同一フォルダに入っている DLL ファイルはすべて実行に必要です。exe だけを別の場所にコピーしないでください。

#### ステップ 2: exe を実行する

`AutoScreenshot.exe` をダブルクリックして起動します。

#### ステップ 3: 起動を確認する

タスクトレイ（通知領域）に緑色のカメラアイコンが表示されれば起動成功です。

### 2.2 ポータブル運用とインストール運用の違い

AutoScreenshot は「ポータブル運用」と「インストール運用」の 2 つの動作モードをサポートします。

| 項目 | ポータブル運用 | インストール運用（既定） |
|---|---|---|
| config.json の場所 | exe と同じフォルダ | `%APPDATA%\AutoScreenshot\config.json` |
| 切り替え方法 | exe と同じフォルダに `config.json` を置く | config.json が exe フォルダにない場合 |
| 使いどき | USB メモリで持ち運ぶ場合、複数環境で設定を共有する場合 |Windows 標準の使い方 |

**ポータブル運用に切り替える手順**:
1. 一度 AutoScreenshot を起動して終了する（`%APPDATA%\AutoScreenshot\config.json` が自動生成される）
2. そのファイルを exe と同じフォルダにコピーする
3. 次回起動から exe フォルダの config.json が優先される

## 3. 初回起動

### SmartScreen の警告が表示された場合

Windows が「このアプリはインターネットからダウンロードされました」として警告を出す場合があります。

1. 「**詳細情報**」をクリックする
2. 「**実行**」をクリックする

> **理由**: 配布元の証明書によるコード署名がないため表示されます。社内配布の場合は グループポリシーで SmartScreen の対象から除外する設定も可能です。

### 初回起動時の動作

- 設定ファイル (`config.json`) が存在しない場合、**デフォルト設定で自動生成**されます
- Windows ログオン時の**自動起動が有効** (`AutoStart: true`) の場合、スタートアップへの登録が行われます
- タスクトレイにアイコンが表示され、すぐに撮影が開始されます

> **初回起動に数秒かかる場合**: `%TEMP%` に .NET ランタイムのリソースファイルが展開されることがあります（初回のみ）。

## 4. 基本操作

### トレイアイコンのメニュー

タスクトレイのアイコンを**右クリック**するとメニューが開きます。

| メニュー項目 | 動作 |
|---|---|
| **一時停止** / **再開** | 撮影を一時停止・再開する。アイコンがグレーになる |
| **保存フォルダを開く** | 既定の保存先をエクスプローラーで開く |
| **設定** | 設定ウィンドウを開く |
| **今すぐ撮影** | 手動でスクリーンショットを撮影する |
| **キャプチャ履歴** | 直近 10 件の撮影ファイルをサブメニューで表示（クリックでエクスプローラーが選択状態で開く） |
| **手順書セッション区切り** | 現在のセッションを保存し、新しいセッションを開始する |
| **手順書を今すぐ生成** | 現在のセッションの手順書を即時生成する（セッションは継続） |
| **動画を生成** | 現在のセッションの動画（APNG/MP4）をバックグラウンドで即時生成する |
| **バージョン情報** | バージョン情報を表示する |
| **終了** | AutoScreenshot を終了する（手順書の最終保存も行う） |

### 撮影トリガー（既定設定）

AutoScreenshot は以下の操作を自動検知して撮影します。

| トリガー | 既定 | 説明 |
|---|---|---|
| マウス左クリック | 有効 | クリック後に撮影 |
| マウス右クリック | 有効 | 右クリック後に撮影 |
| マウス中クリック | 有効 | 中ボタンクリック後に撮影 |
| ドラッグ完了 | 有効 | 200ms 以上の左ボタン押下→離し後に撮影 |
| ホイール操作 | 有効 | ホイール停止 500ms 後に撮影 |
| キーボード入力 | 有効 | 入力アイドル 2 秒後に 1 回撮影 |
| アクティブウィンドウ切替 | 有効 | フォーカス変更時に撮影 |
| 画面差分検知 | 有効 | 3 秒ごとに 30% 以上の変化を検知したとき撮影 |

## 5. 設定

タスクトレイアイコンを右クリック →「**設定**」を選択します。

### 5.1 一般タブ

| 設定項目 | 既定値 | 説明 |
|---|---|---|
| Windows ログオン時に自動起動する | オン | `HKCU\...\Run` にエントリを追加 |
| 一時停止ホットキー | （未設定） | 例: `Ctrl+F9`。テキストボックスをクリックしてキーを押すと登録される |

**ホットキーの設定手順**:
1. 「ホットキー」欄をクリックする
2. 登録したいキー組み合わせを押す（例: `Ctrl+F9`）
3. 欄に「Ctrl+F9」と表示されたことを確認して「適用」をクリックする
4. 解除したい場合は「クリア」ボタンを押す

---

### 5.2 撮影トリガータブ

#### トリガー有効/無効

各トリガー（マウス左/右/中クリック、ドラッグ、ホイール、キーボード、ウィンドウ切替、差分検知）のチェックボックスで個別に有効/無効を切り替えられます。

#### クールダウン設定

同じトリガーが短時間に連続して発生したとき、撮影を間引きます。

| 設定 | 既定値 | 説明 |
|---|---|---|
| クリッククールダウン | 1.0 秒 | クリック連続の間引き |
| ドラッグクールダウン | 0.5 秒 | ドラッグ連続の間引き |
| ホイールクールダウン | 2.0 秒 | ホイール操作の間引き |
| キーボードクールダウン | 2.0 秒 | キーボードの間引き |
| ウィンドウ切替クールダウン | 1.0 秒 | 切替の間引き |
| 差分検知クールダウン | 3.0 秒 | 差分検知の間引き |

#### アイドル・閾値設定

| 設定 | 既定値 | 説明 |
|---|---|---|
| キーボードアイドル待機 | 2.0 秒 | 最終キー入力からこの時間経過後に撮影 |
| ドラッグ判定閾値 | 200 ミリ秒 | 左ボタン押下→離しがこれ以上なければドラッグとみなす |
| ホイールアイドル待機 | 500 ミリ秒 | 最終ホイール操作からこの時間経過後に撮影 |
| 差分検知間隔 | 3 秒 | 差分チェックを行う周期 |
| 差分検知閾値 | 30 % | この割合以上のピクセルが変化したとき撮影 |

### 5.3 保存タブ

| 設定項目 | 既定値 | 説明 |
|---|---|---|
| 保存先フォルダ | `%USERPROFILE%\Pictures\AutoScreenshot` | 「参照」ボタンでフォルダを選択できる |
| 画像形式 | PNG | PNG / JPEG / WebP から選択 |
| JPEG/WebP 画質 | 85 | 0〜100 で指定。値が大きいほど高品質・大容量 |
| フォルダ命名規則 | 日付フォルダ | 下表参照 |
| 低ディスク容量警告しきい値 | 500 MB | 空き容量がこの値を下回るとトースト警告 + 自動一時停止 |

**フォルダ命名規則**:

| 選択肢 | フォルダ構造の例 |
|---|---|
| 日付フォルダ（既定） | `AutoScreenshot\2026-05-27\` |
| 日付＋時間フォルダ | `AutoScreenshot\2026-05-27\14\` |
| セッションフォルダ | `AutoScreenshot\20260527_143022\` |
| フラット（フォルダなし）| `AutoScreenshot\` |

**ファイル命名規則**:

```
{yyyyMMdd_HHmmss_fff}_{トリガー名}_monitor{番号}.{拡張子}

例: 20260527_143022_123_click_monitor1.png
    20260527_143025_456_keyboard_monitor1.png
```

### 5.8 LLM 連携タブ（Azure AI Foundry）

> **前提**: LLM 機能を使用するには、Microsoft Azure AI Foundry で払い出された API キーとエンドポイント URL が必要です。Anthropic の Claude API には直接接続しません。

| 設定項目 | 既定値 | 説明 |
|---|---|---|
| LLM 連携を有効にする | オフ | オンにすると手順書生成時に Azure AI Foundry を呼び出す |
| Azure AI Foundry エンドポイント URL | （空） | 例: `https://<リソース名>.services.ai.azure.com/models` |
| Azure AI Foundry API キー | （空） | Foundry プロジェクトから払い出されたキー |
| デプロイメント名 | `claude-haiku-4-5` | Foundry にデプロイされたモデルのデプロイ名 |

**設定時の注意事項**:

- エンドポイント URL と API キーは入力後、設定保存時に **Windows DPAPI** で暗号化されて `config.json` に保存されます。平文は保存されません
- 暗号化は現在ログインしている Windows ユーザーに紐づくため、**別のユーザーアカウントや別の PC では復号できません**
- 設定変更後は「適用」または「OK」をクリックしてください

**Azure AI Foundry エンドポイント URL の確認方法**:

1. Azure ポータルにサインインする
2. 対象の Azure AI Foundry プロジェクトを開く
3. 「デプロイ」→ 対象のデプロイ名を選択する
4. エンドポイント URL と API キーをコピーする

**LLM が有効な場合の手順書生成の流れ**:

1. セッション終了時（または「手順書を今すぐ生成」時）に全ステップを Azure AI Foundry へ送信
2. Claude が各ステップの説明文をより自然な日本語に改善する（1 セッション = 1 API 呼び出し）
3. 操作内容の 3〜5 行の要約（ダイジェスト）を生成して表紙に記載する
4. 完了後にトースト通知「手順書を生成しました（LLM最適化済み）」が表示される
5. LLM 呼び出しに失敗した場合はルールベースの説明文で手順書を生成し続ける（フォールバック）

### 5.9 動画生成タブ

v1.1.0 で追加されたタブです。AutoScreenshot で記録した操作ステップから動画ファイル（APNG・MP4）を生成します。

#### 出力設定

| 設定項目 | 既定値 | 説明 |
|---|---|---|
| APNG を出力 | オン | アニメーション PNG（`.png`）を生成。ブラウザ・多くのビューアで再生可能 |
| MP4 を出力 | オン | H.264 動画（`.mp4`）を生成。Windows メディアプレーヤーで再生可能 |
| 動画出力先フォルダ | （空 = 自動） | 空欄の場合は保存フォルダ内の `videos\` サブフォルダ |
| 完了時に出力フォルダを開く | オン | 生成完了後にエクスプローラーで出力先を開く |
| 手順書生成と同時に動画を生成 | オフ | オンにすると手順書生成のたびに動画も自動生成する |

#### フレーム設定

| 設定項目 | 既定値 | 説明 |
|---|---|---|
| フレームタイミング | 固定 | **固定**: 全フレームを同じ秒数で表示 / **可変**: TTS 読み上げ時間に合わせる |
| 1フレームの表示秒数（固定時） | 3.0 秒 | 各ステップ画像の表示時間 |
| 最小/最大フレーム秒数（可変時） | 1.0 / 10.0 秒 | TTS 時間をこの範囲でクリップ |
| 出力解像度 | HD (1280×720) | HD / Full HD (1920×1080) / QHD (2560×1440) |
| MP4 ビットレート | 4 Mbps | 値が大きいほど高品質・大容量 |

#### 装飾設定

| 設定項目 | 既定値 | 説明 |
|---|---|---|
| 波紋エフェクト | オン | マウスクリック位置を中心に同心円アニメーションを描画 |
| 操作位置枠 | オン | カーソル周辺に破線矩形を描画 |
| テロップ表示 | オン | フレーム下部に説明文などを表示 |
| テロップ：イベント種別 | オン | `[クリック]` などの操作種別を表示 |
| テロップ：入力テキスト | オン | キーボード入力した文字を表示 |
| テロップ：説明文 | オン | 操作の説明文を表示 |
| テロップ：タイムスタンプ | オン | 操作時刻（HH:mm:ss）を表示 |
| テロップフォントサイズ | 16 pt | テロップの文字サイズ |
| テロップ背景の不透明度 | 160 | 0（透明）〜 255（不透明） |

#### TTS（音声ナレーション）設定

| 設定項目 | 既定値 | 説明 |
|---|---|---|
| TTS 音声ナレーション | オン | 各フレームの説明文を Windows の音声合成で読み上げ、MP4 に合成 |
| 音声 | （システム既定） | インストール済みの SAPI 音声から選択。空欄でシステム既定 |
| 読み上げ速度 | 0 | -10（最遅）〜 10（最速） |
| 音量 | 100 | 0〜100 |

> **注意**: TTS 音声ナレーションは MP4 のみに合成されます。APNG には音声は含まれません。TTS は Windows の音声合成（SAPI）を使用するため、追加インストールは不要ですが、音声の品質はシステム設定に依存します。

## 6. 手順書生成機能の使い方

### 基本的な使い方（ワンオペ）

1. AutoScreenshot を起動する
2. 通常通り PC で作業する（クリック・キー入力を自動で記録）
3. 作業が終わったらタスクトレイ →「**終了**」をクリックする
4. 終了処理中に手順書が自動生成される
5. 保存フォルダ内の `manuals\` フォルダに手順書ファイルが作成される

```
出力ファイル例:
%USERPROFILE%\Pictures\AutoScreenshot\manuals\
  └── 20260527_143022_操作手順書_2026-05-27_14_30.md
```

### セッションを手動で区切る

長い作業を複数の手順書に分けたい場合は、区切りたいタイミングでトレイメニュー →「**手順書セッション区切り**」をクリックします。

- 現在のセッションが保存・完了される
- 新しいセッションが即座に開始される
- 「セッション開始時にタイトルを入力」設定がオンの場合はダイアログが表示される

### 手順書をすぐに確認したい

作業中に暫定版の手順書を確認したい場合は、トレイメニュー →「**手順書を今すぐ生成**」をクリックします。

- 現在のステップを元に手順書が生成される
- セッションは継続される（以降の操作も記録される）

### 手順書の構成

生成される Markdown ファイルの構成:

```markdown
# {セッションタイトル}

- **開始日時**: 2026-05-27 14:30:22
- **終了日時**: 2026-05-27 15:45:10
- **OS / ユーザー**: Microsoft Windows 10.0.26100 / username
- **作成ツール**: AutoScreenshot v1.1.0
- **セッションID**: {GUID}

## 操作内容サマリー
（LLM 有効時のみ。3〜5 行の要約文）

## 目次

1. [1. Google Chrome - メインウィンドウ](#1-google-chrome---メインウィンドウ)
...

## 1. Google Chrome - メインウィンドウ

1. 「検索」をクリックしました。
   ![ステップ 1](../2026-05-27/20260527_143022_click_monitor1.png)

2. 「検索ボックス」に「AutoScreenshot」と入力しました。（キー: A, U, T, O, ...）

### 14:35〜
...
```

### 動画生成機能の使い方 {#61-動画生成機能の使い方}

#### 手動で動画を生成する

1. 操作を記録する（AutoScreenshot が自動でスクリーンショットを撮影）
2. 生成したいタイミングでトレイメニュー →「**動画を生成**」をクリックする
3. バックグラウンドで動画生成が開始される（他の作業を続けられる）
4. 完了するとバルーン通知が表示される
5. 「完了時に出力フォルダを開く」がオンの場合、エクスプローラーが自動で開く

```
出力ファイル例:
%USERPROFILE%\Pictures\AutoScreenshot\videos\
  ├── 20260527_143022_操作手順書_2026_05_27_14_30.png   ← APNG
  └── 20260527_143022_操作手順書_2026_05_27_14_30.mp4   ← MP4
```

#### 手順書生成と同時に動画を生成する

設定「動画生成」タブ →「**手順書生成と同時に動画を生成**」をオンにすると、手順書生成のたびに（セッション終了時・「手順書を今すぐ生成」時）動画も自動生成されます。

#### 動画生成中の挙動

- 動画生成はバックグラウンドで実行されます（UI はブロックされません）
- 同時に複数の動画を生成することはできません（生成中にもう一度「動画を生成」を押しても無視されます）
- 生成中は他のメニュー操作（撮影・手順書生成など）は通常通り使えます

#### 動画が生成されない場合

1. 設定 →「動画生成」タブで「APNG を出力」または「MP4 を出力」が少なくとも一方オンになっているか確認する
2. セッションに有効なスクリーンショットが 1 枚以上あるか確認する（ステップは記録されているがファイルが存在しない場合はスキップされる）
3. ログファイルでエラーを確認する:
   ```
   [ERR] VideoGenerator: 動画生成に失敗しました
   ```
4. TTS が原因の場合は「TTS 音声ナレーション」をオフにして再試行する

---

### 要確認マーカーについて

UIA（UI Automation）と OCR の両方で UI 要素名の取得に失敗したステップには、以下のマーカーが付きます。

```markdown
3. 「座標 (1234, 567)」をクリックしました。 <!-- TODO: UI名を確認してください -->
```

手順書を人手でレビューする際にこのコメントを検索して確認してください。

## 7. 自動起動の設定

### 有効化（推奨）

1. 設定ウィンドウを開く
2. 「一般」タブ →「Windows ログオン時に自動起動する」をオンにする
3. 「OK」をクリックする

以降、Windows ログオン時に自動でバックグラウンド起動します。

### 手動での確認・削除

レジストリへの登録状況を確認・削除する場合:

1. `Win + R` → `regedit` を実行する
2. `HKEY_CURRENT_USER\SOFTWARE\Microsoft\Windows\CurrentVersion\Run` を開く
3. `AutoScreenshot` エントリを確認する

## 8. グローバルホットキー

設定した一時停止ホットキーはアプリがフォーカスを持っていなくても機能します。

**設定手順**（再掲）:
1. 設定 →「一般」タブ →「ホットキー」テキストボックスをクリック
2. 登録したいキー組み合わせを押す（例: `Ctrl+F9`）
3. 「適用」または「OK」をクリック

**推奨ホットキー例**:

| キー | 備考 |
|---|---|
| `Ctrl+F9` | 多くのアプリと競合しにくい |
| `Ctrl+Shift+F10` | さらに競合リスクが低い |
| `Alt+Pause` | キーボードによっては使用できない場合がある |

> **注意**: 他のアプリが同じキーを使用している場合、登録に失敗します（ログに警告が記録されます）。

## 9. アンインストール

### ステップ 1: AutoScreenshot を終了する

### ステップ 2: インストールフォルダを削除する

AutoScreenshot.exe を展開したフォルダをすべて削除します。

3. `AutoScreenshot` エントリを右クリック →「削除」

## 11. トラブルシューティング

### 起動しない / タスクトレイにアイコンが表示されない

**確認事項**:
1. タスクマネージャーで `AutoScreenshot.exe` が起動していないか確認する
   - 起動中の場合: 既に起動している（多重起動の抑止機能により 2 つ目は起動しない）
   - メッセージボックスが「すでに起動しています」と出た場合: タスクトレイの非表示アイコン欄を確認する
2. イベントビューアー（Windows ログ → アプリケーション）でエラーがないか確認する
3. ログファイル `%APPDATA%\AutoScreenshot\logs\app-{日付}.log` を確認する

### 撮影されない

**確認事項**:
1. トレイアイコンが**グレー**になっていないか確認する（グレー = 一時停止中）
2. 対象アプリが「除外アプリ」リストに含まれていないか確認する
3. 設定 →「撮影トリガー」タブで当該トリガーが有効になっているか確認する
4. クールダウン時間が長すぎないか確認する

### 手順書が生成されない

**確認事項**:
1. 設定 →「手順書生成」タブで「手順書生成を有効にする」がオンになっているか確認する
2. 出力先フォルダに書き込み権限があるか確認する
3. ログファイルでエラーが記録されていないか確認する
4. ステップが 0 件の場合は手順書は生成されない（撮影されているか確認する）

### LLM 連携が機能しない

**確認事項**:
1. 設定 →「LLM 連携」タブで「LLM 連携を有効にする」がオンになっているか確認する
2. エンドポイント URL と API キーの**両方**が入力されているか確認する
3. 入力後に「適用」または「OK」をクリックして保存されているか確認する
4. ログファイルで LLM 呼び出しエラーが記録されていないか確認する
   ```
   [WRN] LLM 呼び出し失敗: {endpoint}
   ```
5. Azure AI Foundry のデプロイが有効な状態かポータルで確認する
6. API キーを別のユーザーアカウントや別の PC で設定した設定ファイルを使い回していないか確認する（DPAPI は同一ユーザー・同一 PC でのみ復号できる）

### 動画が生成されない / 生成が途中で止まる

**確認事項**:
1. 設定 →「動画生成」タブで「APNG を出力」または「MP4 を出力」がオンになっているか確認する
2. 記録されたセッションに有効なスクリーンショットファイルが存在するか確認する
3. ログファイルで動画生成エラーを確認する:
   ```
   [ERR] VideoGenerator ...
   [ERR] MfVideoWriter ...
   ```
4. TTS が有効な場合、「TTS 音声ナレーション」をオフにして再試行する（SAPI 音声の問題を切り分け）
5. MP4 生成に失敗する場合、Windows MediaFoundation が有効か確認する（`mf.dll` / `mfreadwrite.dll`）

### パスワード欄がマスクされない

**確認事項**:
1. 設定 →「プライバシー」タブで「パスワード欄を自動マスキング」がオンになっているか確認する
2. 対象アプリが UIAutomation（UIA）をサポートしているか確認する（UIA 非対応アプリはマスク不可）

### ディスク容量の警告が頻繁に出る

1. 設定 →「保存」タブで「低ディスク容量警告しきい値」を下げる（既定: 500 MB）
2. スクリーンショットの保存先フォルダを空き容量の多いドライブに変更する
3. 古いスクリーンショットを定期的に削除する

### 除外アプリを設定したのに撮影される

- プロセス名とウィンドウタイトルの**両方**に対してマッチングします
- 設定したパターンと実際のプロセス名・タイトルが一致しているか確認する
- ログに `IsExcludedApp` の判定結果は出ないため、タスクマネージャーでプロセス名を確認してから設定する
- パターン例: `notepad.exe` ではなく `notepad`（拡張子なし）で指定する

### ログの確認方法

```
ログファイルの場所:
%APPDATA%\AutoScreenshot\logs\app-{yyyyMMdd}.log

主なログレベル:
[INF] - 通常の動作
[WRN] - 警告（機能は継続）
[ERR] - エラー（操作が失敗した可能性）
[DBG] - デバッグ情報（詳細な内部動作）
```
