## 1. 概要と目的

### 1.1 背景

現行の AutoScreenshot（v1.1.0 まで）は、スクリーンショットを撮影するたびに画像ファイルを即座に SaveFolder へ書き出し、セッション終了時に手順書・動画を直接出力する設計になっている。

この方式では以下の課題がある:
- 途中生成物（低品質な中間状態の画像・手順書）がフォルダに散在する
- 「あとから手順書だけ作り直す」「別の解像度で動画を再出力する」といった再エクスポートができない
- 成果物の置き場所（SaveFolder / manuals/ / videos/）が分散しており、1回の操作記録が複数フォルダにまたがる

### 1.2 目的

**プロジェクトフォルダ**を単位として1回の操作記録を一か所にまとめ、その内容からユーザーが任意のタイミングで成果物（画像・手順書・動画・ZIPアーカイブ）を**エクスポート**できる仕組みを追加する。

### 1.3 ヒアリング結果サマリー

| 項目 | 選択内容 |
|---|---|
| プロジェクト形式 | **フォルダ形式**（`{name}.ascproj/`）|
| プロジェクト単位 | **ManualSession 単位**（現行のセッション区切りと同じ）|
| 画像格納 | プロジェクト内 `images/` に即時保存 ＋ `thumbs/` にサムネイル別保存 |
| エクスポート成果物 | 個別画像（PNG/JPEG）・手順書（Markdown/Word）・動画（APNG/MP4）・ZIPアーカイブ |
| 既存フォルダとの関係 | `SaveFolder` 直下にプロジェクトフォルダを作成（既存の日付フォルダに替えて）|
| プロジェクト管理UI | **専用プロジェクトビューウィンドウ**を新設 |
| ステップ編集 | **削除** ＋ **説明文の手修正**を可能にする |
| ファイル関連付け | なし（エクスポートはトレイメニューまたはプロジェクトビューから実施）|

## 2. 用語定義

| 用語 | 定義 |
|---|---|
| **プロジェクト** | 1回のManualSessionに対応する記録単位。プロジェクトフォルダとして永続化される |
| **プロジェクトフォルダ** | 拡張子 `.ascproj` を持つフォルダ。画像・メタデータ・エクスポート済みファイルを内包 |
| **project.json** | プロジェクトフォルダ内のメタデータファイル。セッション情報・ステップ一覧・エクスポート履歴を保持 |
| **エクスポート** | project.json とimages/の内容から成果物（手順書・動画・画像）を生成し、exports/フォルダへ書き出す操作 |
| **プロジェクトビュー** | 過去のプロジェクト一覧表示・ステップ閲覧・エクスポート操作を行う専用ウィンドウ |

## 3. プロジェクトフォルダ仕様

### 3.1 フォルダ構造

```
{SaveFolder}/
  ├── {YYYYMMDD_HHmmss}_{titleSlug}.ascproj/    ← プロジェクトフォルダ
  │   ├── project.json      ← メタデータ（セッション情報・ステップ一覧）
  │   ├── images/           ← 撮影画像（原寸・元フォーマット）
  │   │   ├── step_001_click_monitor1.png
  │   │   ├── step_002_keyboard_monitor1.png
  │   │   └── ...
  │   ├── thumbs/           ← サムネイル（自動生成・JPEG・最大 320px）
  │   │   ├── step_001.jpg
  │   │   ├── step_002.jpg
  │   │   └── ...
  │   └── exports/          ← エクスポート済みファイル（任意作成）
  │       ├── 20260527_143022_手順書.md
  │       ├── 20260527_143022_手順書.docx
  │       ├── 20260527_143022.png          ← APNG
  │       └── 20260527_143022.mp4
  │
  └── {YYYYMMDD_HHmmss}_{titleSlug}.ascproj/    ← 別プロジェクト
      └── ...
```

**フォルダ名規則:**
- `{YYYYMMDD_HHmmss}_{titleSlug}.ascproj`
- `titleSlug`: セッションタイトルから記号・スペースを `_` に置換し、最大 40 文字に切り詰め
- 例: `20260527_143022_操作手順書_2026_05_27_14_30.ascproj`

### 3.2 project.json スキーマ

```json
{
  "formatVersion": "1.0",
  "projectId": "9c2cafeb-3f9f-4ea1-b0cc-16de29a26402",
  "title": "操作手順書 2026-05-27 14:30",
  "createdAt": "2026-05-27T14:30:22.000+09:00",
  "lastModifiedAt": "2026-05-27T15:45:10.000+09:00",
  "osInfo": "Microsoft Windows 10.0.26100 / username",
  "endedAt": "2026-05-27T15:45:10.000+09:00",
  "digest": "（LLM生成ダイジェスト、null可）",
  "steps": [
    {
      "stepNumber": 1,
      "timestamp": "2026-05-27T14:30:25.123+09:00",
      "triggerType": "MouseLeftClick",
      "uiElementName": "検索ボタン",
      "uiControlType": "Button",
      "cursorX": 640,
      "cursorY": 480,
      "windowTitle": "Google Chrome",
      "processName": "chrome",
      "inputText": null,
      "keyCodes": null,
      "imagePath": "images/step_001_click_monitor1.png",
      "thumbPath": "thumbs/step_001.jpg",
      "descriptionRuleBased": "「検索ボタン」をクリックしました。",
      "descriptionLlm": "検索ボタンをクリックして検索を実行しました。",
      "descriptionOverride": null,
      "needsReview": false,
      "isDeleted": false
    }
  ],
  "exportHistory": [
    {
      "exportedAt": "2026-05-27T16:00:00.000+09:00",
      "type": "Markdown",
      "outputPath": "exports/20260527_160000_手順書.md"
    }
  ]
}
```

**フィールド説明:**

| フィールド | 型 | 説明 |
|---|---|---|
| `formatVersion` | string | project.json のスキーマバージョン（将来の互換性管理用） |
| `projectId` | string(GUID) | プロジェクト一意識別子 |
| `descriptionOverride` | string? | ユーザーが手修正した説明文。非null の場合は LLM・ルールベースより優先 |
| `isDeleted` | bool | ステップ削除フラグ。`true` のステップはエクスポート対象から除外。画像ファイルは `images/_deleted/` に移動 |
| `exportHistory` | array | エクスポート実行ログ（型・日時・出力パス） |

## 4. 機能要件

### FR-PJ01: プロジェクトの自動作成

| ID | FR-PJ01 |
|---|---|
| 概要 | ManualSession 開始時にプロジェクトフォルダを自動作成する |
| トリガー | `ManualSessionRecorder.StartSession()` 呼び出し時 |
| 処理 | 1. `{SaveFolder}/{YYYYMMDD_HHmmss}_{titleSlug}.ascproj/` フォルダを作成<br>2. `images/`・`thumbs/` サブフォルダを作成<br>3. 初期 `project.json` を書き込む（steps=[], exportHistory=[]）|
| 設定 | `config.json` の `storage.saveFolder` を SaveFolder として使用 |
| 制約 | フォルダ名に使えない文字（`<>:"/\|?*`）は `_` に置換する |

---

### FR-PJ02: 撮影時のプロジェクト更新

| ID | FR-PJ02 |
|---|---|
| 概要 | スクリーンショット撮影ごとに images/ へ保存し project.json を追記更新する |
| トリガー | `FileStorage.SaveAsync()` 完了後 |
| 処理 | 1. 画像ファイルを `{project}/images/{filename}` に保存<br>2. サムネイル（最大 320px 幅、JPEG Q=75）を `{project}/thumbs/{stepN}.jpg` に生成<br>3. project.json の steps に新 ManualStep を追記する |
| 原子性 | project.json の書き込みは `{file}.tmp` に書いてからリネームし、途中終了でファイルが破損しない |
| 既存動作との差分 | 画像の保存先が `{SaveFolder}/{date}/` から `{project}/images/` に変わる。それ以外の撮影ロジックは変更なし |

---

### FR-PJ03: project.json の更新（ステップ記録）

| ID | FR-PJ03 |
|---|---|
| 概要 | `ManualSessionRecorder.RecordStepAsync()` 完了後に project.json を更新する |
| 処理 | UIA/OCR 取得後の ManualStep を project.json の steps 配列末尾に追記（JSON マージパッチ方式ではなく全体書き直し） |
| 注意 | 高頻度撮影時（数百ms間隔）でも整合性を保つため、書き込みは排他ロック（`lock` または `SemaphoreSlim`）で保護 |

---

### FR-PJ04: エクスポート — 個別画像ファイル

| ID | FR-PJ04 |
|---|---|
| 概要 | プロジェクトの非削除ステップの画像を exports/ に書き出す |
| 出力 | `{project}/exports/{YYYYMMDD_HHmmss}_images/{ステップ連番}_{trigger}.png` |
| 変換 | 出力フォーマット（PNG/JPEG/WebP）は ManualGenConfig.ImageFormat に従う |
| フィルタ | `isDeleted=true` のステップは除外 |

---

### FR-PJ05: エクスポート — 手順書（Markdown/Word）

| ID | FR-PJ05 |
|---|---|
| 概要 | プロジェクトの内容から Markdown/.docx 手順書を生成し exports/ に書き出す |
| 出力 | `{project}/exports/{YYYYMMDD_HHmmss}_{titleSlug}.md` / `.docx` |
| 説明文優先順位 | `descriptionOverride` → `descriptionLlm` → `descriptionRuleBased` の順 |
| 画像パス | 手順書内の相対パスは `../images/` からの相対参照 |
| LLM | エクスポート時に LLM 有効設定の場合、未処理ステップに対して LLM 改善を実行し project.json を更新してからエクスポート |
| エクスポート後 | exportHistory に記録 |

---

### FR-PJ06: エクスポート — 動画（APNG/MP4）

| ID | FR-PJ06 |
|---|---|
| 概要 | プロジェクトの内容から動画を生成し exports/ に書き出す |
| 出力 | `{project}/exports/{YYYYMMDD_HHmmss}_{titleSlug}.png` / `.mp4` |
| 入力 | project.json の非削除ステップの `imagePath` を VideoGenerator に渡す |
| 既存との差分 | `VideoGenerator.GenerateAsync()` の呼び出し元を ManualSession から Project 経由に変更する以外、生成ロジックは変更なし |
| エクスポート後 | exportHistory に記録 |

---

### FR-PJ07: エクスポート — ZIP アーカイブ

| ID | FR-PJ07 |
|---|---|
| 概要 | プロジェクトフォルダ全体（exports/ を除く）を ZIP に圧縮して書き出す |
| 出力 | ユーザー指定のパス（ファイル保存ダイアログで選択）|
| 包含内容 | `project.json`・`images/`（非削除ステップのみ）・`thumbs/` |
| 除外 | `exports/` フォルダ（成果物は別管理）|
| 用途 | 別 PC でプロジェクトを開く / バックアップ |

---

### FR-PJ08: プロジェクトビューウィンドウ

| ID | FR-PJ08 |
|---|---|
| 概要 | 過去のプロジェクト一覧とステップ閲覧・編集・エクスポートを行う専用ウィンドウを提供する |

**ウィンドウ構成:**

```
[プロジェクトビュー]                                    [×]
┌──────────────────────────────────────────────────────────┐
│ プロジェクト一覧（左ペイン）  │ ステップ詳細（右ペイン） │
│                               │                           │
│ ▶ 20260527_143022 操作手順書  │ [サムネイル大]            │
│   20260527_100000 セットアップ│ ステップ 3 / 12           │
│   20260526_090000 定例業務    │ [クリック] 14:30:25       │
│   ...                         │ 「検索ボタン」をクリック  │
│                               │ [説明文編集テキストボックス]│
│   ─────────────────────       │                           │
│   [プロジェクトフォルダを開く]│ [ ← ] [ → ]  [削除]      │
│                               │                           │
│                               │ ─────────────────────     │
│                               │ エクスポート:             │
│                               │ [画像] [手順書] [動画] [ZIP]│
└──────────────────────────────────────────────────────────┘
```

**左ペイン（プロジェクト一覧）:**
- SaveFolder 直下の `.ascproj` フォルダを検索して一覧表示
- 表示順: 作成日時の降順
- 表示内容: タイトル・作成日時・ステップ数（非削除）・最終エクスポート日時
- 選択でサムネイル一覧に切り替え

**右ペイン（ステップ詳細）:**
- サムネイル一覧表示（GridView、isDeleted=true のステップは薄く表示）
- ステップ選択で拡大表示
- 説明文の表示・手修正（テキストボックスで編集後、確定で project.json を更新）
- 「削除」ボタン: isDeleted を true にし、画像を `images/_deleted/` に移動
- ← / → ナビゲーション

**エクスポートセクション:**
- [画像] ボタン → FR-PJ04 を実行
- [手順書] ボタン → FR-PJ05 を実行（Markdown/Word の選択ダイアログ付き）
- [動画] ボタン → FR-PJ06 を実行（バックグラウンド実行、進捗バルーン）
- [ZIP] ボタン → FR-PJ07 を実行（ファイル保存ダイアログ）

### FR-PJ09: ステップ削除

| ID | FR-PJ09 |
|---|---|
| 概要 | プロジェクトビューからステップを削除する |
| 処理 | 1. `isDeleted = true` に更新して project.json を保存<br>2. 画像ファイルを `images/_deleted/` に移動（物理削除はしない）<br>3. サムネイルを `thumbs/_deleted/` に移動<br>4. UI のステップ一覧をリフレッシュ（ステップ番号は変わらない）|

### FR-PJ11: トレイメニューの変更

現行のトレイメニューを以下のように変更する。

**変更前（v1.1.0）:**
```
手順書セッション区切り
手順書を今すぐ生成
動画を生成
```

**変更後（v1.2.0）:**
```
プロジェクト区切り（新しいプロジェクトを開始）
─────────────────────
エクスポート >
    手順書を生成（現在のプロジェクト）
    動画を生成（現在のプロジェクト）
    画像を書き出す（現在のプロジェクト）
─────────────────────
プロジェクトを管理...
```

| メニュー項目 | 動作 |
|---|---|
| **プロジェクト区切り** | 現在のプロジェクト（セッション）を閉じて新しいプロジェクトを開始（旧「手順書セッション区切り」相当）|
| **エクスポート > 手順書を生成** | 現在のプロジェクトから手順書を即時生成 → exports/ へ |
| **エクスポート > 動画を生成** | 現在のプロジェクトから動画を即時生成 → exports/ へ（バックグラウンド）|
| **エクスポート > 画像を書き出す** | 現在のプロジェクトの全画像を exports/images/ へ一括コピー |
| **プロジェクトを管理...** | FR-PJ08 のプロジェクトビューウィンドウを開く |

## 6. 非機能要件

| ID | 要件 | 実装方針 |
|---|---|---|
| NF-PJ01 | 撮影パフォーマンスへの影響最小化 | project.json の書き込みは `Task.Run` で非同期実行。撮影メインスレッドをブロックしない |
| NF-PJ02 | project.json の耐障害性 | `.tmp` ファイルに書いてからリネーム（原子的置換）。中断でも直前の状態が保持される |
| NF-PJ03 | サムネイル生成の非同期化 | サムネイル生成は撮影直後に `Task.Run` で並行実行。失敗しても撮影は継続 |
| NF-PJ04 | 大量ステップ時の project.json サイズ | 1 ステップあたり約 500 bytes として、1000 ステップ ≈ 500 KB 程度。問題なし |
| NF-PJ05 | プロジェクトビューの応答性 | サムネイル一覧は `VirtualizingStackPanel` による遅延読み込みで実現 |
| NF-PJ06 | 既存 SaveFolder との共存 | `.ascproj` 拡張子フォルダ以外のファイル・フォルダには手を加えない |
| NF-PJ07 | エクスポートの多重実行防止 | エクスポート中は同一プロジェクトへの再エクスポート操作を無効化（ボタン・メニューをグレーアウト）|

## 7. 既存機能との変更差分

### 7.1 撮影・保存

| 項目 | v1.1.0（変更前） | v1.2.0（変更後） |
|---|---|---|
| 画像保存先 | `{SaveFolder}/{date}/{filename}` | `{project}/images/{filename}` |
| サイドカーログ（.log/.jsonl） | `{SaveFolder}/{date}/events_{date}.log` | `{project}/project.json` に統合（events.log は廃止も可、設定で維持可）|
| フォルダ命名規則 | DateWithTimestamp / DateHour / Session / Flat | 廃止（プロジェクトフォルダ名に統一）|

> **設定互換**: `FolderNamingRule` 設定は残すが、プロジェクト機能が有効な場合は無視する。

### 7.2 手順書・動画の生成タイミング

| 項目 | v1.1.0 | v1.2.0 |
|---|---|---|
| 手順書生成 | セッション終了時 / 今すぐ生成 | **エクスポート**時（任意のタイミング）|
| 動画生成 | 「動画を生成」メニュー時 | **エクスポート**時（任意のタイミング）|
| 出力先 | `{OutputFolder}/` / `{SaveFolder}/videos/` | `{project}/exports/` |

### 7.3 削除・廃止する処理

| 処理 | 廃止理由 |
|---|---|
| `FileStorage.SaveAsync()` での直接 SaveFolder 書き込み | プロジェクト内 images/ に変更 |
| `ManualSessionRecorder.WriteSessionAsync()` での即時手順書生成（終了時） | AutoExportOnClose 設定で制御（互換オプション as 既定オン）|

## 8. 後方互換性・移行方針

### 8.1 既存の画像フォルダ

`{SaveFolder}/2026-05-27/` 等の旧形式フォルダはそのまま残す。プロジェクトビューには表示されない（`.ascproj` フォルダのみ対象）。新規撮影はすべてプロジェクトフォルダへ。

### 8.2 config.json の互換性

v1.2.0 の config.json に `project` セクションを追加。v1.1.0 以前の config.json を読み込んだ場合はデフォルト値（プロジェクト機能有効・AutoExportOnClose 手順書オン）で補完する。

### 8.3 フォールバックモード

設定「プロジェクト機能を有効にする」をオフにすると、v1.1.0 以前の動作（SaveFolder への即時書き出し）に戻る。ただし、このフォールバックモードは将来のバージョンで廃止する予定。

### 8.4 既存 ManualSession データの移行

既存の ManualSession（メモリ上のみ）をプロジェクトに変換するための移行ツールは v1.2.0 スコープ外。必要であれば v1.3 以降で対応する。
