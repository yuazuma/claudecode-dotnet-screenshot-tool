## 1. FR-H1: before 画像フォーマットを after 画像と統一

### 1.1 概要

v1.6.0 で実装した before 画像は PNG 固定だった。
v1.7.0 では、before 画像のフォーマットを **after 画像の設定（PNG / JPEG / WebP）と揃える**。
ただし before 画像は証跡用途のため、画像の劣化を最小化する方針とする。

### 1.2 変更仕様

| 設定フォーマット | before 画像の保存形式 | 圧縮パラメーター |
|---|---|---|
| **PNG** | PNG | 変更なし（可逆・劣化なし） |
| **JPEG** | JPEG | **quality = 100**（無圧縮を明示。量子化ロス最小化） |
| **WebP** | WebP | **lossless = true**（可逆 WebP。after の品質設定を使用しない）|

いずれのフォーマットでも before 画像の劣化を最小化・排除する。

### 1.3 影響箇所

| クラス / ファイル | 変更内容 |
|---|---|
| `Services/TriggerOrchestrator.cs` | `FireBeforeCapture()`: `Encode(bmp, ImageFormat.Png)` → `Encode(bmp, cfg.Storage.ImageFormat, beforeJpegQuality: 100)` |
| `Services/FileStorage.cs` | `SaveBeforeAsync()`: ファイル拡張子を形式に合わせて変更（固定の `.png` を廃止） |
| `Services/CaptureService.cs` | `Encode()` の JPEG 呼び出しで quality=100 を明示できるか確認（既存の `jpegQuality` 引数を使用） |
| `Models/TriggerConfig.cs` | `CaptureBeforeImage` プロパティのコメントを更新（「PNG 固定」表記を削除） |

### 1.4 非機能要件

| ID | 要件 |
|---|---|
| NF-H101 | JPEG で保存する場合は `quality = 100` を明示し、他のエクスポート処理の `JpegQuality` 設定値（既定 85）を参照しない |
| NF-H101b | WebP で保存する場合は `lossless = true` を明示し、他のエクスポート処理の `JpegQuality` 設定値を参照しない |
| NF-H102 | 既存の before 画像（PNG で保存済み）はそのまま保持する（再変換不要） |
| NF-H103 | `FileStorage.SaveBeforeAsync()` が返すパスの拡張子は実際に保存したフォーマットと一致させる |

### 1.5 ヒアリング事項

- [x] **WebP 形式の場合は lossless WebP で保存する** → 確定

## 2. FR-H2: after 画像もロスレス保存に変更

### 2.1 概要

FR-H1 で before 画像をロスレス保存に変更したのと同様に、
**after 画像（トリガーキャプチャで保存されるメイン画像）もロスレス保存**とする。

すべての撮影画像（before / after 共通）を証跡として劣化なしで保持する方針に統一する。

### 2.2 変更仕様

FR-H1 と同一のロスレスポリシーを after 画像にも適用する:

| 設定フォーマット | after 画像の保存形式 | 圧縮パラメーター |
|---|---|---|
| **PNG** | PNG | 変更なし（可逆・変更不要） |
| **JPEG** | JPEG | **quality = 100**（無圧縮を明示） |
| **WebP** | WebP | **lossless = true**（可逆 WebP） |

### 2.3 影響箇所

| クラス / ファイル | 変更内容 |
|---|---|
| `Services/TriggerOrchestrator.cs` | `FireCapture()` の `Encode(bmp, cfg.Storage.ImageFormat, cfg.Storage.JpegQuality)` → ロスレス設定に変更 |
| `Services/CaptureService.cs` | `Encode()` に JPEG quality=100 / lossless WebP の呼び出しパスを追加（または引数追加） |

### 2.4 スコープ外（変更しない）

| 対象 | 理由 |
|---|---|
| サムネイル（`ThumbnailService`） | プレビュー用途であり高品質は不要 |
| エクスポート時の画像コピー（`ExportService`） | ディスク上の元ファイルをそのまま参照するため再エンコードなし |
| `StorageConfig.JpegQuality` 設定項目 | 設定値はサムネイル等の他用途で参照される可能性があるため廃止しない |

### 2.5 非機能要件

| ID | 要件 |
|---|---|
| NF-H201 | after 画像エンコード時に `cfg.Storage.JpegQuality`（既定 85）を参照しない |
| NF-H202 | PNG / JPEG / WebP いずれの設定でも、撮影された after 画像は劣化なしで保存される |

## 3. FR-H3: 保存先フォルダの階層構造テンプレート設定

### 3.1 概要

画像・手順書・動画のすべての保存先を、以下の3階層で構成できるようにする。

```
{ベースフォルダ} / {日付テンプレートフォルダ} / {個々のファイル}
```

日付テンプレートフォルダの名前は、セッション開始日時を基準とした
**テンプレート文字列**で指定する。

### 3.2 ヒアリング回答サマリー

| 項目 | 回答 |
|---|---|
| 日時の基準 | **セッション開始日時** |
| `.ascproj` フォルダへの適用 | **適用する**（プロジェクトフォルダ名もテンプレート化） |
| 既存フォルダ命名規則との関係 | **完全に置き換える**（`FolderNaming` enum 廃止） |
| 出力タイプ別設定 | **画像 / 手順書 / 動画 それぞれ独立したテンプレート** |

### 3.3 テンプレート仕様

#### 使用可能なトークン

テンプレート文字列は **日時トークン** と **プレースホルダー** と **リテラル文字** を組み合わせる。

##### 日時トークン（.NET `DateTime.ToString()` 書式指定子）

セッション開始日時（`sessionStart`）を `ToString()` で変換する。

| トークン | 意味 | 変換例（2026-05-30 14:30:22） |
|---|---|---|
| `yyyy` | 4桁年 | `2026` |
| `MM` | 2桁月 | `05` |
| `dd` | 2桁日 | `30` |
| `HH` | 2桁時（24時間） | `14` |
| `mm` | 2桁分 | `30` |
| `ss` | 2桁秒 | `22` |

##### プレースホルダー（`{…}` 形式）

`{…}` をプレースホルダーとして認識し、日時トークンより先に展開してから
`DateTime.ToString()` に渡す。

**日時短縮形:**

| プレースホルダー | 展開後 | 変換例（2026-05-30 14:30:22） |
|---|---|---|
| `{date}` | `yyyyMMdd` | `20260530` |
| `{datetime}` | `yyyyMMddHHmmss` | `20260530143022` |
| `{date_time}` | `yyyyMMdd_HHmmss` | `20260530_143022` |
| `{date-time}` | `yyyyMMdd-HHmmss` | `20260530-143022` |
| `{hour}` | `HH00` | `1400` |

**セッション情報:**

| プレースホルダー | 内容 | 変換例 |
|---|---|---|
| `{title}` | セッション / プロジェクトのタイトル（最大100文字） | `操作手順書_2026_05_30_14_30` |
| `{title_short}` | タイトルを40文字でカット | `操作手順書_2026_05_30` |
| `{id}` | セッション ID の先頭8文字（衝突防止） | `a1b2c3d4` |

**環境情報:**

| プレースホルダー | 内容 | 変換例 |
|---|---|---|
| `{username}` | Windows ログオンユーザー名 | `y` |
| `{computername}` | コンピューター名 | `DESKTOP-ABC123` |

##### テンプレート評価の順序

1. `{…}` プレースホルダーを対応する文字列（または日時書式文字列）に展開する
2. 展開後の文字列を `sessionStart.ToString(expanded, CultureInfo.InvariantCulture)` で評価する
3. ファイルシステムで使用できない文字（`/` `\` `:` `*` `?` `"` `<` `>` `|`）を `_` に置換する

**テンプレート例:**

| テンプレート文字列 | 変換結果 |
|---|---|
| `proj-{datetime}` | `proj-20260530143022` |
| `yyyyMMdd-HH00-顧客定例会` | `20260530-1400-顧客定例会` |
| `{date_time}_{title_short}.ascproj` | `20260530_143022_操作手順書.ascproj` |
| `{date}` | `20260530` |
| `{date-time}_{username}` | `20260530-143022_y` |
| `{datetime}_{id}` | `20260530143022_a1b2c3d4` |
| `（空文字）` | フォルダ階層なし（ベースフォルダ直下にファイル） |

#### `.ascproj` フォルダ名テンプレート

プロジェクト機能が有効の場合、日付テンプレートフォルダが `.ascproj` フォルダを兼ねる。
テンプレートに `.ascproj` 拡張子を含めることで、プロジェクトフォルダとして機能する。

```
{画像ベースフォルダ} / {ProjectFolderTemplate} / images / {ファイル}
                                                 / thumbs / {ファイル}
                                                 / exports / {ファイル}
```

テンプレートに `.ascproj` を含めない場合も、システムが自動的に付加する（後方互換のため）。

### 3.4 設定項目

> **既定値の方針**: 画像・手順書・動画のベースフォルダは、すべて
> `Environment.GetFolderPath(SpecialFolder.MyPictures) + "\AutoScreenshot"`
> （= `C:\Users\{ユーザー名}\Pictures\AutoScreenshot`）を共通の既定値とする。
> 個別に変更することは可能。

#### 画像（Images）

| 設定項目 | 型 | 説明 | 既定値 |
|---|---|---|---|
| `ImageBaseFolder` | string | ベースフォルダ（絶対パス） | `C:\Users\{username}\Pictures\AutoScreenshot` |
| `ImageFolderTemplate` | string | 日付テンプレート（非プロジェクト時） | `yyyyMMdd` |
| `ProjectFolderTemplate` | string | プロジェクトフォルダ名テンプレート | `{date_time}_{title_short}.ascproj` |

> **移行**: 旧 `StorageConfig.SaveFolder` → `ImageBaseFolder`、`FolderNaming` enum → 廃止

#### 手順書（Manual）

| 設定項目 | 型 | 説明 | 既定値 |
|---|---|---|---|
| `ManualBaseFolder` | string | ベースフォルダ（絶対パス） | `C:\Users\{username}\Pictures\AutoScreenshot` |
| `ManualFolderTemplate` | string | 日付テンプレート | `{date_time}` |

> **移行**: 旧 `ManualGenConfig.OutputFolder` → `ManualBaseFolder`

#### 動画（Video）

| 設定項目 | 型 | 説明 | 既定値 |
|---|---|---|---|
| `VideoBaseFolder` | string | ベースフォルダ（絶対パス） | `C:\Users\{username}\Pictures\AutoScreenshot` |
| `VideoFolderTemplate` | string | 日付テンプレート | `{date_time}` |

> **移行**: 旧 `VideoGenConfig.VideoOutputFolder` → `VideoBaseFolder`

### 3.5 実装方針

#### テンプレート評価

セッション開始時に `DateTime.Now` を取得し、`sessionStart.ToString(template)` で評価する。
ただし `.ascproj` 等の拡張子部分は日付変換対象外としてリテラルとして扱う。

```csharp
// テンプレート評価関数（例）
string EvaluateTemplate(string template, DateTime sessionStart)
    => sessionStart.ToString(template, CultureInfo.InvariantCulture);
```

#### ベースフォルダとの結合

```
finalPath = Path.Combine(baseFolder, EvaluateTemplate(template, sessionStart), fileName)
```

テンプレートが空文字の場合は中間フォルダなし:
```
finalPath = Path.Combine(baseFolder, fileName)
```

| クラス / ファイル | 変更内容 |
|---|---|
| `Models/AppConfig.cs` | `StorageConfig`: `SaveFolder`→`ImageBaseFolder`、`FolderNaming`廃止、`ImageFolderTemplate`・`ProjectFolderTemplate` 追加 |
| `Models/AppConfig.cs` | `ManualGenConfig`: `OutputFolder`→`ManualBaseFolder`、`ManualFolderTemplate` 追加 |
| `Models/VideoGenConfig.cs` | `VideoOutputFolder`→`VideoBaseFolder`、`VideoFolderTemplate` 追加 |
| `Services/FileStorage.cs` | `BuildFolderPath()` をテンプレート評価方式に変更 |
| `Services/ProjectStore.cs` | `CreateProjectAsync()` でプロジェクトフォルダ名にテンプレートを使用 |

### 3.7 非機能要件

| ID | 要件 |
|---|---|

## 4. FR-H4: 設定ウィンドウへの「パス設定」タブ追加

### 4.1 概要

パス・フォルダ関連の設定項目が複数タブ（「保存」「手順書生成」「動画生成」）に
分散しているため、比較・設定がしにくい。**専用の「パス設定」タブ**を追加し、
全出力タイプのパス設定を1か所に集約する。

### 4.2 移動対象の設定項目

| 元のタブ | 移動する設定項目 |

| 「手順書生成」タブ | 出力先フォルダ（ベースフォルダ）、フォルダテンプレート |
| 「動画生成」タブ | 動画出力フォルダ（ベースフォルダ）、フォルダテンプレート、エクスプローラーで開く |

### 4.3 「パス設定」タブの UI レイアウト（案）


#### ツールチップのテキスト（案）

```

  yyyy=年　MM=月　dd=日　HH=時　mm=分　ss=秒

  {date}      yyyyMMdd の短縮形

  {title}     セッションタイトル
  {title_short} セッションタイトル（40文字まで）
  {id}        セッションID先頭8文字（衝突防止）
  {username}  ログオンユーザー名
  {computername} コンピューター名

【使用例】
  {date_time}_{title_short}.ascproj
  → 20260530_143022_操作手順書.ascproj
  yyyyMMdd-HH00-顧客定例会
  → 20260530-1400-顧客定例会
```

元のタブ（「保存」「手順書生成」「動画生成」）からパス関連の設定項目のみを削除し、
残りの設定はそれぞれのタブに残す。

### 4.5 機能要件

## 5. FR-H5: 第2ベースフォルダへのフォールバック保存

### 5.1 概要

第1ベースフォルダが存在するドライブにアクセスできない、または空き容量不足などで
書き込みに失敗した場合に、**第2ベースフォルダ**に自動的に切り替えて保存を継続する。

フォールバックは以下の2つのタイミングで発動する:
1. **アプリ起動時** — 第1ベースフォルダのドライブ存在確認・書き込みテスト
2. **撮影中** — 実際の書き込み失敗（例外）を検知した瞬間に即座に切り替え

### 5.2 設定項目

出力タイプごとに独立した第2ベースフォルダを設定する（FR-H3 の設計と統一）。

#### 画像（Images）への追加

| 設定項目 | 型 | 説明 | 既定値 |
|---|---|---|---|
| `ImageFallbackBaseFolder` | string | 画像の第2ベースフォルダ（空 = フォールバックなし） | `（空文字）` |

#### 手順書（Manual）への追加

| 設定項目 | 型 | 説明 | 既定値 |
|---|---|---|---|
| `ManualFallbackBaseFolder` | string | 手順書の第2ベースフォルダ（空 = フォールバックなし） | `（空文字）` |

#### 動画（Video）への追加

| 設定項目 | 型 | 説明 | 既定値 |
|---|---|---|---|
| `VideoFallbackBaseFolder` | string | 動画の第2ベースフォルダ（空 = フォールバックなし） | `（空文字）` |

フォールバック先が空文字の場合はフォールバックなし（失敗ログのみ）。
フォルダテンプレートは第1・第2ともに同じものを使用する。

### 5.3 起動時ドライブチェック

アプリ起動時（`NotifyIconWrapper.Initialize()`）に第1ベースフォルダを確認する。

**チェック項目:**
1. **ドライブ存在確認** — `DriveInfo(drivePath).IsReady`
2. **書き込みテスト** — 一時ファイルを作成・削除して書き込み権限を確認

**チェック失敗時の動作:**
1. 第2ベースフォルダが設定されていれば即座に切り替え
2. バルーン通知で切り替わったことをユーザーに通知
3. ログに `[WRN]` レベルで記録

```
例: バルーン通知テキスト
「保存先ドライブ（D:\）に接続できません。
  第2ベースフォルダ（C:\Users\y\Pictures\AutoScreenshot）に切り替えました。」
```

### 5.4 撮影中の書き込みエラー検知とフォールバック

`FileStorage.SaveAsync()` / `SaveBeforeAsync()` 内で `IOException` または
`UnauthorizedAccessException` が発生した場合:

1. 第2ベースフォルダが設定済みかつまだ切り替えていない → **即座に切り替え**
2. 失敗したファイルを第2ベースフォルダに再保存
3. バルーン通知でユーザーに通知
4. 以降の保存はすべて第2ベースフォルダへ（セッション終了まで）

```
例: バルーン通知テキスト
「保存先への書き込みに失敗しました。
  第2ベースフォルダ（C:\Users\y\Pictures\AutoScreenshot）に切り替えました。」
```

### 5.5 フォールバック後の挙動

| 項目 | 仕様 |
|---|---|
| フォールバック後の保存先 | 第2ベースフォルダを使用（同じテンプレートを適用） |
| 第1への自動復旧 | **しない**。セッション終了まで第2ベースフォルダを使い続ける |
| 次回起動時 | 改めてドライブチェックを行い、第1が復旧していれば第1を使用 |
| プロジェクト内の `images/` 等サブフォルダ | 第2ベースフォルダ配下の `.ascproj` に継続保存 |

### 5.6 ディスク容量不足の検知

既存の `LowDiskSpaceThresholdMb` 設定（既定 500MB）を下回った場合:
- 従来: バルーン通知のみ
- v1.7.0: **第2ベースフォルダへの自動フォールバック**も実行する

## 6. FR-H6: エクスポート進捗ダイアログ

### 6.1 概要

手順書・動画・画像・ZIP のエクスポート処理中に、**ノンモーダルの進捗ウィンドウ**を表示する。
ユーザーは処理中もアプリを操作でき、進捗を確認しながらキャンセルもできる。

### 6.2 ヒアリング回答サマリー

| 項目 | 回答 |
|---|---|
| ダイアログ種別 | **ノンモーダル**（アプリ操作を妨げない） |
| キャンセル | **可能**（途中で中断できる） |
| 対象エクスポート | **全種別**（手順書 MD/Word/HTML・動画・画像・ZIP） |
| 表示トリガー | **両方**（ProjectViewWindow のボタン + トレイメニューの自動エクスポート） |

### 6.3 進捗ウィンドウの UI

```
┌─ エクスポート — AutoScreenshot ──────────────────────── [×] ┐
│  [アイコン] Markdown 手順書を生成中...                       │
│                                                              │
│  [████████████░░░░░░░░░░░░░░░░░] 48%  ステップ 60 / 125    │
│                                                              │
│  ファイル: exports\20260530_143022_操作手順書.md             │
│                                                              │
│                                   [キャンセル]               │
└──────────────────────────────────────────────────────────────┘
```

#### 表示項目

| 要素 | 内容 |
|---|---|
| タイトルバー | `エクスポート — AutoScreenshot` |
| 操作説明ラベル | 現在実行中の操作名（例: `Markdown 手順書を生成中...`） |
| プログレスバー | 確定値（0〜100%） or 不確定（Indeterminate）を切り替え |
| カウンターラベル | `ステップ N / 合計M`（ステップ処理がある場合） |
| ファイルパスラベル | 出力先ファイルパス（長い場合は末尾を省略表示） |
| キャンセルボタン | クリックでキャンセルリクエスト送信 |

#### エクスポート種別ごとの進捗単位

| エクスポート | プログレスバー | カウンター |
|---|---|---|
| Markdown / Word | 確定（ステップ数で計算） | `ステップ N / M` |
| HTML | 確定（ステップ数で計算） | `ステップ N / M` |
| 動画（APNG / MP4） | 確定（フレーム数で計算） | `フレーム N / M` |
| 画像を書き出す | 確定（ファイル数で計算） | `ファイル N / M` |
| ZIP | 不確定（Indeterminate） | `ファイル N 個追加済み` |

#### ウィンドウの位置・動作

- 初回表示位置: 画面右下（タスクトレイ付近）
- `Topmost = false`（他ウィンドウの背後に隠れることを許容）
- 複数エクスポートが同時進行する場合: 各エクスポートに1ウィンドウ（重ねて表示）
- 完了時: 自動的に閉じる（操作不要）
- キャンセル時: `キャンセルしました` とラベルを更新し、2秒後に自動で閉じる

### 6.4 キャンセル機能

- キャンセルボタンを押すと `CancellationTokenSource.Cancel()` を呼び出す
- エクスポートメソッドはステップ間で `ct.ThrowIfCancellationRequested()` を確認する
- キャンセル後に生成途中のファイルが存在する場合は削除しない（不完全なファイルを残す）
  - ユーザーが手動で確認・削除できるようにするため
- `OperationCanceledException` は進捗ウィンドウでキャッチしてラベルを更新し、ログに `[INF]` で記録する

### 6.5 進捗報告の実装方針

`IProgress<ExportProgress>` インターフェースを使用して、エクスポートサービスから UI スレッドに進捗を通知する。

```csharp
public record ExportProgress(
    string OperationName,    // 例: "Markdown 手順書を生成中..."
    int Current,             // 現在のステップ/フレーム数
    int Total,               // 合計ステップ/フレーム数（0 = Indeterminate）
    string? OutputPath       // 出力先パス（null = 非表示）
);
```

エクスポートメソッドのシグネチャに `IProgress<ExportProgress>?` と `CancellationToken` を追加する:

```csharp
// 変更例（ExportService）
public async Task ExportMarkdownAsync(
    ProjectInfo project,
    IProgress<ExportProgress>? progress = null,
    CancellationToken ct = default);
```

### 6.6 トレイメニュー自動エクスポートへの対応

セッション終了時の自動エクスポート（`ManualSessionRecorder.WriteSessionAsync()`）でも
進捗ウィンドウを表示する。

- `Application.Current.Dispatcher.Invoke()` で WPF ウィンドウを UI スレッドで生成・表示
- 自動エクスポートでもキャンセル可能とする（キャンセル時はエクスポートをスキップ）

### 6.7 トレイアイコン ツールヒントへの進捗表示

エクスポート実行中は、トレイアイコンのツールヒント（`NotifyIcon.Text`）に
**現在の進捗を追記**する形で表示する。

#### 表示形式

```
通常時:  AutoScreenshot - 本日 42 枚撮影
エクスポート中:
         AutoScreenshot - 本日 42 枚撮影
         MD 60/125 生成中...
```

複数エクスポートが同時進行する場合は改行して並べる:

```
AutoScreenshot - 本日 42 枚撮影
MD 60/125 生成中...
MP4 32/125 生成中...
```

#### 種別ごとのラベル（短縮形）

| エクスポート種別 | ツールヒント内ラベル |
|---|---|
| Markdown 手順書 | `MD N/M 生成中...` |
| Word 手順書 | `DOCX N/M 生成中...` |
| HTML 手順書 | `HTML N/M 生成中...` |
| 動画（APNG） | `APNG N/M 生成中...` |
| 動画（MP4） | `MP4 N/M 生成中...` |
| 画像エクスポート | `画像 N/M 書き出し中...` |
| ZIP | `ZIP 作成中...`（不確定のため N/M なし） |

#### 制約

- `NotifyIcon.Text` の上限は **127 文字**。進捗行が長くなる場合は末尾を省略する
- 完了またはキャンセル後はツールヒントを通常表示（進捗行なし）に戻す
- 更新頻度は `IProgress<ExportProgress>` の `Report()` 呼び出しごと（ステップ間）

#### 実装

`Notifier.SetExportProgress(IReadOnlyList<ExportProgress> active)` を新規追加し、
進捗ウィンドウの進捗更新と同時に呼び出す。アクティブなエクスポートが
0 件になれば自動的に通常テキストに戻す。

### 6.9 ProjectViewWindow のステータスバーとの関係

現行の不確定プログレスバー（`PbStatus`）は引き続き維持するが、
エクスポート詳細の進捗は新しい進捗ウィンドウに委ねる。
`PbStatus` はエクスポート中に表示し続け（ウィンドウ全体の「ビジー」表示として機能）、
詳細は進捗ウィンドウで確認する。
