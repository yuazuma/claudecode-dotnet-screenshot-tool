## 1. 概要と目的

### 1.1 背景

v1.2.0 でプロジェクトファイル機能が導入され、1 回の操作記録が `.ascproj/` フォルダに一元管理されるようになった。v1.3.0 では以下 3 つの課題を解消する：

1. **手順書の共有コストが高い** — Word / Markdown はビューアや環境依存がある。ブラウザで完結する HTML 形式が欲しい
2. **LLM 処理の待機が長い** — セッション終了時に全ステップをまとめて LLM に投げるため、長時間セッションでは数分待つ必要がある
3. **手順書の視認性が低い** — 操作箇所（矢印・番号）がなく、スクリーンショットを見ただけでは「どこを押したか」がわかりにくい

また v1.2.0 のプロジェクト管理 UI をより実用的にするため、タグ付け・検索・ステップ並び替え・手動挿入を追加し、複数セッションをまたいだ手順書作成を可能にするプロジェクト結合・分割機能を導入する。

### 1.2 バージョンポリシー

- **バージョン番号**: 1.3.0
- **後方互換性**: v1.2.0 の `project.json`（formatVersion 1.0）を読み込める。新フィールドは既定値で補完
- **設定互換**: `config.json` の既存エントリは変更しない。v1.3.0 追加設定は新キーで追記

## 3. FR-A: HTML エクスポート

### 3.1 概要

プロジェクトの非削除ステップから、**自己完結型 HTML ファイル**（スクリーンショットを Base64 埋め込み）を生成する。ブラウザで開くだけで手順書を参照でき、社内ポータル・Teams などへのリンク共有にも適している。

### 3.2 出力仕様

| 項目 | 内容 |
|---|---|
| 出力先 | `{project}/exports/{YYYYMMDD_HHmmss}_{titleSlug}.html` |
| 文字コード | UTF-8（BOM なし） |
| 外部依存 | なし（CSS・画像はすべてインライン） |
| 印刷対応 | `@media print` CSS で改ページ制御 |

#### HTML 構造

```html
<!DOCTYPE html>
<html lang="ja">
<head>
  <meta charset="UTF-8">
  <title>{プロジェクトタイトル}</title>
  <style>
    /* インライン CSS: 目次・ステップカード・印刷スタイル */
  </style>
</head>
<body>
  <h1>{プロジェクトタイトル}</h1>
  <p>{作成日} — {ステップ数}ステップ</p>
  <!-- サマリー（LLM Digest があれば表示） -->
  <nav id="toc">
    <ol>
      <li><a href="#step-1">ステップ 1: {説明文冒頭30文字}</a></li>
      ...
    </ol>
  </nav>
  <!-- ステップカード -->
  <section id="step-1" class="step">
    <h2>ステップ 1</h2>
    <p class="meta">{triggerType} — {timestamp} — {windowTitle}</p>
    <img src="data:image/png;base64,{base64}" alt="ステップ1">
    <p class="desc">{EffectiveDescription}</p>
  </section>
  ...
</body>
</html>
```

#### 説明文優先順位

`descriptionOverride` → `descriptionLlm` → `descriptionRuleBased`（v1.2.0 と同じ）

#### アノテーション対応（FR-C と連携）

FR-C（ステップアノテーション）が実装されている場合、Base64 に焼き込んだアノテーション済み画像を使用する。

### 3.3 機能要件

| ID | 要件 |
|---|---|
| FR-A01 | `HtmlManualWriter.WriteAsync(ProjectInfo, outputPath)` を実装する |
| FR-A02 | 画像を `Convert.ToBase64String` で埋め込む。画像が存在しないステップはプレースホルダーを表示 |
| FR-A03 | `ExportService.ExportHtmlAsync(ProjectInfo)` から呼び出す |
| FR-A04 | ProjectViewWindow に「手順書 (HTML)」ボタンを追加する |
| FR-A05 | トレイメニューの「エクスポート >」サブメニューに「HTML 手順書を生成」を追加する |
| FR-A06 | エクスポート完了後、`exportHistory` に `Type: "Html"` で記録する |
| FR-A07 | `ProjectConfig.AutoExportHtml: bool`（既定: オフ）を追加し、セッション終了時の自動エクスポート対象にする |

### 3.4 CSS 設計方針

- ページ幅: 最大 900px、中央寄せ
- ステップカード: 1ステップ = 1 `<section>`、枠線・影付き
- サムネイルは幅 100%（最大 800px）で表示
- 目次は左上に固定（`position: sticky`）または先頭に配置
- `@media print`: 各ステップで改ページ、背景色なし

## 4. FR-B: インクリメンタル LLM 処理

### 4.1 概要

現行（v1.2.0）はセッション終了時に全ステップを一括で LLM に投げる。v1.3.0 では **ステップ記録のたびに LLM を非同期で呼び出し**、説明文をリアルタイムで project.json に反映する。ユーザーは記録中に ProjectViewWindow で説明文の品質を確認・修正できる。

### 4.2 設計

#### 処理フロー

```
RecordStepAsync()
  └─ RecordProjectStepAsync()  ← 既存：project.json 追記
       └─ [Fire-and-forget] EnqueueLlmStepAsync(pStep)
            └─ LlmQueue（SemaphoreSlim で 1 並列に制限）
                 └─ LlmService.ImproveStepAsync(step)
                      └─ step.DescriptionLlm を更新
                           └─ ProjectStore.WriteProjectJsonAsync()
```

#### LlmQueue の設計

| 項目 | 内容 |
|---|---|
| 並列数 | 1（`SemaphoreSlim(1,1)`）。同時リクエストでレート制限にかかるのを防ぐ |
| キューサイズ | 無制限（`ConcurrentQueue<ProjectStep>`）|
| キュードレイン | セッション終了（`StopSessionAsync`）時に最大 60 秒待機してキューが空になるまで処理を継続 |
| LLM 失敗時 | `descriptionLlm` は null のまま。次ステップの処理を継続する（エラーは `Log.Warning` のみ）|

#### LlmService の変更

新メソッドを追加する。既存の `ImproveDescriptionsAsync(ManualSession)` は変更しない（後方互換）。

```csharp
/// <summary>単一ステップの説明文を LLM で改善して返す。</summary>
public Task<string?> ImproveStepDescriptionAsync(string triggerType, string? uiElementName,
    string? windowTitle, string? ruleBasedDescription, CancellationToken ct = default);
```

プロンプト: 既存の `ImproveDescriptionsAsync` で使っているシステムプロンプトを流用し、1 ステップ分だけ送る。

### 4.3 機能要件

| ID | 要件 |
|---|---|
| FR-B01 | `ManualSessionRecorder` に `LlmStepQueue`（`ConcurrentQueue<(ProjectInfo, ProjectStep)>`）を追加する |
| FR-B02 | ステップ記録後、LLM 設定が有効な場合はキューに追加して非同期処理を開始する |
| FR-B03 | キュー処理は `SemaphoreSlim(1,1)` で直列化し、API レート制限を回避する |
| FR-B04 | `LlmService.ImproveStepDescriptionAsync()` を新規追加する |
| FR-B05 | LLM 完了後、`ProjectStep.DescriptionLlm` を更新して `ProjectStore.WriteProjectJsonAsync()` を呼ぶ |
| FR-B06 | `StopSessionAsync` / `SplitSession` では、既存の一括 LLM 処理（`ImproveDescriptionsAsync`）を**スキップ**し、キュードレイン（最大 60 秒待機）に移行する |
| FR-B07 | インクリメンタル LLM が有効な場合、ProjectViewWindow はポーリング（1 秒間隔）で project.json を再読み込みして説明文を更新表示する |
| FR-B08 | `ProjectConfig.IncrementalLlm: bool`（既定: オン、LLM 有効設定の場合のみ動作）を追加する |

### 4.4 ProjectViewWindow のポーリング更新

- ProjectViewWindow を開いている間、選択中のプロジェクトを 1 秒ごとに再読み込みする `DispatcherTimer` を追加する
- LLM 処理中のステップはステップラベルに「(処理中...)」を表示する（`DescriptionLlm == null && !step.IsDeleted` の場合）
- 説明文 TextBox にフォーカスがある場合はポーリング更新を一時停止する（編集中の内容が上書きされないようにする）

## 5. FR-C: ステップアノテーション

### 5.1 概要

ProjectViewWindow のステップ詳細エリアで、選択ステップの画像上に**番号バッジ・矢印・ハイライト矩形・吹き出し**を追加できる。アノテーションは `project.json` の `ProjectStep.Annotations` に保存し、エクスポート時（Markdown / Word / HTML / 動画フレーム）に画像へ焼き込む。

### 5.2 アノテーションデータモデル

```csharp
public class AnnotationItem
{
    public string Type { get; set; } = ""; // "Number" | "Arrow" | "Rect" | "Callout"
    public int X { get; set; }             // 画像座標（原寸 px）
    public int Y { get; set; }
    public int X2 { get; set; }            // Arrow: 先端 X / Rect: 右端 X / それ以外: 未使用
    public int Y2 { get; set; }            // Arrow: 先端 Y / Rect: 下端 Y / それ以外: 未使用
    public string? Label { get; set; }     // Number の数値文字列 / Callout のテキスト
    public string Color { get; set; } = "#FF0000"; // hex カラー
}
```

`ProjectStep` に `List<AnnotationItem>? Annotations` を追加する（null = アノテーションなし）。

### 5.3 ProjectViewWindow のアノテーション編集 UI

ステップ詳細エリアの右側に「アノテーション」ペインを追加する。

```
[ステップ画像（Canvas オーバーレイ付き）]
────────────────────────────────
ツール: [番号] [矢印] [矩形] [吹き出し]
カラー: [■赤] [■青] [■黄] [■緑]
[クリア]  [確定（保存）]
```

- WPF `Canvas` を `Image` の上にオーバーレイして描画
- 各アノテーション種別のクリック/ドラッグ操作:

| 種別 | 操作 |
|---|---|
| 番号バッジ | クリックした位置に `①②③...` の丸数字を追加 |
| 矢印 | ドラッグ開始点 → 終点へ矢印を引く |
| 矩形ハイライト | ドラッグで矩形を描く（半透明塗り + 実線枠）|
| 吹き出し | クリックでテキスト入力ダイアログを表示、吹き出し図形で配置 |

- 確定ボタンで `ProjectStep.Annotations` を更新して `project.json` を保存する
- 「クリア」ボタンで全アノテーションを削除する

### 5.4 アノテーション焼き込み処理

`AnnotationRenderer` クラスを新規追加する。`System.Drawing.Graphics` で画像上に描画し、新しい PNG として返す。

```csharp
public static class AnnotationRenderer
{
    /// <summary>アノテーション済み画像を返す（元ファイルは変更しない）。</summary>
    public static Bitmap Render(string imagePath, IEnumerable<AnnotationItem> annotations);
}
```

**エクスポート時の適用箇所:**

| エクスポート種別 | 適用方法 |
|---|---|
| HTML | Base64 変換前に `AnnotationRenderer.Render()` を適用 |
| 画像エクスポート (FR-PJ04) | アノテーション済み画像をコピー先に書き出す |
| Markdown / Word | 手順書内の画像参照を一時的なアノテーション済み画像に差し替え |
| 動画フレーム | `FrameRenderer` の画像入力としてアノテーション済み Bitmap を渡す |

### 5.5 機能要件

| ID | 要件 |
|---|---|
| FR-C01 | `Models/AnnotationItem.cs` を新規追加する |
| FR-C02 | `ProjectStep.Annotations: List<AnnotationItem>?` を追加する（null safe、既存プロジェクトは null） |
| FR-C03 | `Services/AnnotationRenderer.cs` を新規追加する |
| FR-C04 | ProjectViewWindow に Canvas オーバーレイとツールパレットを追加する |
| FR-C05 | 確定操作で project.json を保存する |
| FR-C06 | `ExportService` の全エクスポートパス（HTML/画像/Markdown/Word）でアノテーション焼き込みを適用する |
| FR-C07 | アノテーションが空 (`null` または空リスト) の場合は元画像をそのまま使用する（パフォーマンス考慮） |

## 6. FR-D: プロジェクト管理強化

### 6.1 タグ付け

`ProjectInfo` に `List<string> Tags` を追加する（既定: 空リスト）。

**ProjectViewWindow 左ペインの変更:**

```
[検索: _____________] [×クリア]
タグ: [業務手順] [設定] [テスト] [+追加]
────────────────────────────────
▶ 20260527 操作手順書  [業務手順]
  20260526 システム設定 [設定]
  ...
```

- タグ一覧はすべてのプロジェクトのタグをユニーク集計して表示
- タグをクリックでフィルタリング（複数選択 AND 条件）
- 右クリックメニューまたはプロジェクト詳細エリアでタグの追加・削除

### 6.2 プロジェクト検索

- 左ペイン上部に検索ボックスを追加
- タイトル・ステップの説明文・タグに対して大文字小文字を区別しないインクリメンタル検索
- 検索ボックスが空の場合は全件表示

### 6.3 ステップ並び替え

- `LstSteps`（サムネイルグリッド）でドラッグ＆ドロップによるステップ順序変更
- 並び替え後、`StepNumber` を 1 からの連番に再採番し project.json を保存する
- サムネイルファイル名（`step_NNN.jpg`）は更新しない（`ThumbPath` の参照で管理）

### 6.4 ステップ手動挿入

- 「＋ステップを追加」ボタンを ProjectViewWindow に追加
- 説明文（テキスト）と画像ファイル（省略可）を指定してステップを挿入する
- 挿入位置: 現在の選択ステップの後
- 画像を指定した場合は `images/` にコピーし、サムネイルを生成する

### 6.5 機能要件

| ID | 要件 |
|---|---|
| FR-D01 | `ProjectInfo.Tags: List<string>` を追加する |
| FR-D02 | ProjectViewWindow 左ペインに検索ボックスとタグフィルタを追加する |
| FR-D03 | `ProjectStore.ListProjectsAsync()` に検索文字列・タグフィルタの引数を追加する（後方互換: デフォルト引数） |
| FR-D04 | `LstSteps` に `AllowDrop="True"` を設定し、ドラッグ＆ドロップで並び替えを実現する |
| FR-D05 | 並び替え後に `ProjectStep.StepNumber` を再採番して project.json を保存する |
| FR-D06 | 「＋ステップを追加」ボタンを追加し、`ProjectStep` を手動で作成・挿入する |
| FR-D07 | 手動挿入ステップは `TriggerType = "Manual"` とする |

## 7. FR-E: プロジェクト結合・分割

### 7.1 プロジェクト結合（マージ）

複数のプロジェクトを時系列順に結合して新しいプロジェクトを生成する。

**操作**: ProjectViewWindow 左ペインで複数プロジェクトを選択（Ctrl+クリック）→「結合...」ボタン

**仕様:**

| 項目 | 内容 |
|---|---|
| 出力 | 新しい `.ascproj/` フォルダ（新 ProjectId）|
| タイトル | ダイアログで入力（既定: 最初のプロジェクトのタイトル） |
| ステップ | 入力プロジェクトのステップを `createdAt` 昇順で並べ、`StepNumber` を 1 から再採番 |
| 画像 | 入力プロジェクトの `images/` から新プロジェクトの `images/` にコピー |
| サムネイル | 新 `StepNumber` に合わせて再生成 |
| エクスポート履歴 | 結合後プロジェクトは空（`exportHistory = []`）|
| 元プロジェクト | 変更しない（元の `.ascproj/` フォルダはそのまま残る）|

```csharp
// ProjectStore に追加
public Task<ProjectInfo> MergeProjectsAsync(IEnumerable<ProjectInfo> sources, string newTitle);
```

### 7.2 プロジェクト分割（スプリット）

既存プロジェクトの指定ステップ以降を別プロジェクトに切り出す。

**操作**: ステップを選択 →「ここで分割」ボタン

**仕様:**

| 項目 | 内容 |
|---|---|
| 元プロジェクト | 指定ステップ **以前**（ステップ番号を再採番）|
| 新プロジェクト | 指定ステップ **以降**（ステップ番号を 1 から再採番）|
| タイトル | ダイアログで 2 つのタイトルを入力 |
| 画像 | 各プロジェクトに必要な画像のみをコピー（元の `images/` からコピー）|
| 元プロジェクトフォルダ | 分割後も残す（ユーザーが手動で削除）|

```csharp
// ProjectStore に追加
public Task<(ProjectInfo before, ProjectInfo after)> SplitProjectAsync(
    ProjectInfo source, int splitAtStepNumber, string titleBefore, string titleAfter);
```

### 7.3 機能要件

| ID | 要件 |
|---|---|
| FR-E01 | `ProjectStore.MergeProjectsAsync()` を実装する |
| FR-E02 | `ProjectStore.SplitProjectAsync()` を実装する |
| FR-E03 | ProjectViewWindow 左ペインで Ctrl+クリックによる複数選択を有効にする |
| FR-E04 | 「結合...」ボタンを左ペインに追加する（複数選択時のみ有効）|
| FR-E05 | 「ここで分割」ボタンをステップ詳細エリアに追加する |
| FR-E06 | 結合・分割後は ProjectViewWindow のプロジェクト一覧をリフレッシュする |

```
    画像を書き出す（現在のプロジェクト） ← 既存
```

## 9. 非機能要件

| ID | 要件 | 実装方針 |
|---|---|---|
| NF-A01 | HTML エクスポートの生成時間 | 100 ステップ以下で 5 秒以内。画像サイズが大きい場合は非同期で進捗通知 |
| NF-B01 | インクリメンタル LLM のレート制限対応 | キュー直列化（1 並列）。失敗時は指数バックオフなしで単純にスキップ |
| NF-B02 | セッション終了時の LLM キュードレイン | 最大 60 秒待機。タイムアウト後は未処理ステップを `descriptionLlm = null` のまま残す |
| NF-C01 | アノテーション描画の精度 | 原寸座標で保存し、表示時にスケーリングして Canvas 上に描画する |
| NF-C02 | アノテーション焼き込みは元画像を変更しない | 必ずコピーまたはメモリ上の Bitmap に描画する |
| NF-D01 | ステップ並び替えの一貫性 | 並び替え直後に project.json を同期保存し、UI と整合を取る |
| NF-E01 | マージ・スプリットは元プロジェクトを変更しない | 新しいフォルダを作成し、ファイルはコピーする |
