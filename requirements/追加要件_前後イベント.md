## 1. 概要と目的

### 1.1 背景

v1.5.x まで、クリック・キーボードなどの操作イベントに対してスクリーンショットは
**操作後（ボタンアップ）** のタイミングで1枚のみ取得していた。

このため手順書・証跡として「**操作前の画面状態**」が記録されず、以下の課題があった：

- 「どの画面でそのボタンを押したか」を示す前画面がなく、手順書として不完全
- UI の変化（ダイアログ表示・ページ遷移・入力確定など）を前後で比較できない
- 証跡用途では「操作時点の元画面」が必須だが現在は記録されない

### 1.2 目的

1 つの操作ステップに対して「**操作前 (before)** 」と「**操作後 (after)** 」の
2 枚のスクリーンショットを取得・保存し、手順書と証跡の双方の品質を高める。

### 1.3 設計上の前提

| 項目 | 方針 |
|---|---|
| **操作前画像のフォーマット** | PNG 固定（劣化なし・再利用性を優先） |
| **操作前画像へのアノテーション** | 付与しない（証跡として元画像のまま保管） |
| **操作後画像** | 既存の設定（PNG / JPEG / WebP）に従う。アノテーション付与可 |
| **データモデル** | 命名規則・標準的な実装として `BeforeImagePath` / `AfterImagePath` に統一する |
| **後方互換性** | v1.5.x の project.json を移行してロードできる（旧 `imagePath` → `afterImagePath` に自動変換） |

### 1.4 バージョンポリシー

- **バージョン番号**: 1.6.0
- **後方互換**: v1.5.x の `project.json`（`imagePath` / `thumbPath` フィールド）をロード時に
  `afterImagePath` / `afterThumbPath` へ自動移行する
- **設定互換**: `config.json` の既存エントリは変更しない。v1.6.0 追加設定は新キーで追記

## 3. FR-F1: 操作前スクリーンショット取得

### 3.1 概要

マウスボタン押下イベント（DOWN）またはキーボードシーケンス開始時に、
アプリが操作を処理する**前**のスクリーンショットを取得して PNG で保存する。

### 3.2 トリガー別の "before" タイミング

| トリガー | before 取得タイミング | 備考 |
|---|---|---|
| 左クリック | `WM_LBUTTONDOWN` | ドラッグ判定より先に撮影 |
| 右クリック | `WM_RBUTTONDOWN` | 現行の after より前 |
| 中クリック | `WM_MBUTTONDOWN` | 現行の after より前 |
| ドラッグ&ドロップ | `WM_LBUTTONDOWN`（ドラッグ開始前） | after はドロップ完了後（変更なし） |
| キーボード | キーシーケンス開始（アイドルタイマー未稼働時の最初のキー） | after は `KeyboardIdleSeconds` 経過後（変更なし） |
| アクティブウィンドウ切替 | **非対象**（単一スナップショットで十分） | — |
| 画面差分 | **非対象**（差分検知自体が変化後を示す） | — |
| 手動撮影 | **非対象**（意図的な1枚取得） | — |

### 3.3 撮影仕様

| 項目 | 内容 |
|---|---|
| フォーマット | **PNG 固定**（グローバルの `ImageFormat` 設定に関わらず） |
| マスキング | `MaskPasswordFields` が有効な場合はパスワード欄を黒塗り（プライバシー保護のため before も適用） |
| カーソルオーバーレイ | **適用しない**（before はクリック前の素の画面状態） |
| タイムスタンプ焼き込み | **適用しない**（アノテーションと同じく証跡の改変を避ける） |
| マルチモニター | 全モニターを撮影（after と同じ） |

### 3.4 機能要件

| ID | 要件 |
|---|---|
| FR-F101 | `HookService` に `MouseBeforeEvent` イベント（`EventHandler<TriggerType>`）を追加し、`WM_LBUTTONDOWN` / `WM_RBUTTONDOWN` / `WM_MBUTTONDOWN` で発火する |
| FR-F102 | `HookService` のキーボードフックで、`_keyboardIdleTimer` が未稼働（タイマーが null または完了済み）の状態で最初のキーが押下されたとき `KeyboardBeforeEvent` を発火する |
| FR-F103 | `TriggerOrchestrator` が `MouseBeforeEvent` / `KeyboardBeforeEvent` を受信したら `FireBeforeCapture` を Task.Run で非同期実行する。フック コールバックをブロックしない |
| FR-F104 | `FireBeforeCapture` はスクリーンショットを PNG で保存し、結果のパスを `_pendingBeforeShots[triggerType]` に格納する |
| FR-F105 | `_pendingBeforeShots` は `Dictionary<TriggerType, (string path, DateTime capturedAt)>` とし、ロック付きで排他制御する |
| FR-F106 | before 撮影は `CaptureBeforeImage` 設定（後述）が有効な場合のみ実行する |
| FR-F107 | 除外アプリ (`ExcludeApps`) チェックは before / after 共通で適用する |

## 4. FR-F2: 操作後スクリーンショット取得（遅延撮影）

### 4.1 概要

ボタンアップイベント後に `PostClickDelayMs` ミリ秒待機してから after スクリーンショットを取得する。
これによりターゲットアプリが操作を処理して画面を更新するまでの時間を確保する。

### 4.2 after 取得タイミングの変更

| トリガー | 変更前 | 変更後 |
|---|---|---|
| 左クリック | `WM_LBUTTONUP` 直後に撮影 | `WM_LBUTTONUP` + `PostClickDelayMs` ms 後に撮影 |
| 右クリック | `WM_RBUTTONDOWN` 直後に撮影 | `WM_RBUTTONUP` + `PostClickDelayMs` ms 後に撮影 |
| 中クリック | `WM_MBUTTONDOWN` 直後に撮影 | `WM_MBUTTONUP` + `PostClickDelayMs` ms 後に撮影 |
| ドラッグ&ドロップ | `WM_LBUTTONUP`（閾値超え）直後 | 同 + `PostClickDelayMs` ms 後 |
| キーボード | `KeyboardIdleSeconds` 経過後（変更なし） | 変更なし |
| ホイール | `WheelIdleMs` 経過後（変更なし） | 変更なし |

右クリックのタイミング変更（DOWN → UP）は before 分離のために必要な変更でもある。

### 4.3 機能要件

| ID | 要件 |
|---|---|
| FR-F201 | `HookService` の右クリック処理を `WM_RBUTTONDOWN`（発火）→ `WM_RBUTTONUP` + 遅延（発火）に変更する |
| FR-F202 | `HookService` の中クリック処理を `WM_MBUTTONDOWN`（発火）→ `WM_MBUTTONUP` + 遅延（発火）に変更する |
| FR-F203 | 左クリック・ドラッグの after 撮影に `PostClickDelayMs` の遅延を追加する |
| FR-F204 | 遅延は `Task.Delay` ではなく `System.Threading.Timer`（ワンショット）で実装し、フックコールバックをブロックしない |
| FR-F205 | `TriggerOrchestrator.FireAfterCapture` で after を保存後、`_pendingBeforeShots[triggerType]` から before パスを取り出し、`ManualSessionRecorder.RecordStepAsync` に `beforeImagePath` / `afterImagePath` の両方を渡す |
| FR-F206 | `_pendingBeforeShots` に対応する before がない場合（`CaptureBeforeImage = false` 等）は `beforeImagePath = null` として問題なく動作させる |
| FR-F207 | `PostClickDelayMs` は `TriggerConfig` の設定値として参照する（後述） |

## 5. FR-F3: データモデル変更

### 5.1 命名規則の整理

v1.5.x まで `ImagePath` / `ThumbPath` という命名だった。
v1.6.0 では操作前後の 2 種類の画像を保持するため、
命名の対称性・明示性を優先して以下のとおり統一する。

| v1.5.x | v1.6.0 | 意味 |
|---|---|---|
| `ImagePath` | `AfterImagePath` | 操作後の画像（メイン画像・アノテーション対象） |
| `ThumbPath` | `AfterThumbPath` | 操作後画像のサムネイル |
| —（新規） | `BeforeImagePath` | 操作前の画像（PNG 固定・証跡） |
| —（新規） | `BeforeThumbPath` | 操作前画像のサムネイル |

### 5.2 ManualStep

```csharp
public class ManualStep
{
    public int StepNumber { get; set; }
    public DateTime Timestamp { get; set; }
    public TriggerType TriggerType { get; set; }
    public string? UiElementName { get; set; }
    public string? UiControlType { get; set; }
    public System.Drawing.Point CursorPosition { get; set; }
    public string WindowTitle { get; set; } = "";
    public string ProcessName { get; set; } = "";
    public string? InputText { get; set; }
    public string? KeyCodes { get; set; }

    /// <summary>操作後スクリーンショットのフルパス（設定フォーマット）。アノテーション付与可。</summary>
    public string? AfterImagePath { get; set; }

    /// <summary>操作前スクリーンショットのフルパス（PNG 固定・証跡）。アノテーション付与不可。</summary>
    public string? BeforeImagePath { get; set; }

    public string DescriptionRuleBased { get; set; } = "";
    public string? DescriptionLlm { get; set; }
    public bool NeedsReview { get; set; }
}
```

### 5.3 ProjectStep

```csharp
public class ProjectStep
{
    public int StepNumber { get; set; }
    public DateTimeOffset Timestamp { get; set; }
    public string TriggerType { get; set; } = "";
    public string? UiElementName { get; set; }
    public string? UiControlType { get; set; }
    public int CursorX { get; set; }
    public int CursorY { get; set; }
    public string WindowTitle { get; set; } = "";
    public string ProcessName { get; set; } = "";
    public string? InputText { get; set; }
    public string? KeyCodes { get; set; }

    /// <summary>images/ 配下への相対パス（操作後・設定フォーマット）。アノテーション対象。</summary>
    public string? AfterImagePath { get; set; }

    /// <summary>thumbs/ 配下への相対パス（操作後画像のサムネイル）。</summary>
    public string? AfterThumbPath { get; set; }

    /// <summary>images/before/ 配下への相対パス（操作前・PNG 固定・証跡）。</summary>
    public string? BeforeImagePath { get; set; }

    /// <summary>thumbs/before/ 配下への相対パス（操作前画像のサムネイル）。</summary>
    public string? BeforeThumbPath { get; set; }

    public string DescriptionRuleBased { get; set; } = "";
    public string? DescriptionLlm { get; set; }
    public string? DescriptionOverride { get; set; }
    public bool NeedsReview { get; set; }
    public bool IsDeleted { get; set; }
    public List<AnnotationItem>? Annotations { get; set; }

    [JsonIgnore]
    public string EffectiveDescription =>
        DescriptionOverride ?? DescriptionLlm ?? DescriptionRuleBased;
}
```

### 5.4 JSON 後方互換（project.json の移行）

`ProjectStore.ReadProjectJsonAsync` 内でデシリアライズ後に移行処理を行う。

```csharp
// v1.5.x の project.json では "imagePath" / "thumbPath" フィールドが使われていた。
// v1.6.0 ではこれらを afterImagePath / afterThumbPath に自動移行する。
// JsonDocument で旧フィールドを読み取り、新フィールドが null の場合に設定する。
foreach (var step in info.Steps)
{
    // ReadProjectJsonAsync の後処理として実装する（ProjectStep に旧フィールドの
    // 互換プロパティを設けることで System.Text.Json の通常デシリアライズで対応してもよい）
}
```

移行後に `WriteProjectJsonAsync` を呼んだタイミングで `afterImagePath` として永続化される。
旧 `imagePath` フィールドは v1.6.0 以降の書き込みでは出力されない。

具体的な実装方針（2 案を提示。実装時に選択）：

**案 A（JsonPropertyName シム）**: `ProjectStep` に旧フィールド名でデシリアライズ専用プロパティを追加。

```csharp
// シリアライズ対象外（WhenWritingNull で null → 出力スキップ）
[JsonPropertyName("imagePath")]
public string? LegacyImagePath
{
    get => null;           // 書き込み時は常に null → 出力されない
    init => AfterImagePath ??= value;   // 読み込み時は AfterImagePath に設定
}

[JsonPropertyName("thumbPath")]
public string? LegacyThumbPath
{
    get => null;
    init => AfterThumbPath ??= value;
}
```

**案 B（ProjectStore で移行）**: `ReadProjectJsonAsync` で `JsonDocument` を使って旧フィールドを検出し、デシリアライズ後に `AfterImagePath` / `AfterThumbPath` へコピーする。

いずれの案でも既存 project.json は無変更でロードできる。

### 5.5 TriggerConfig への追加

```csharp
public class TriggerConfig
{
    // ... 既存フィールド ...

    /// <summary>クリック前スクリーンショットを取得する。</summary>
    public bool CaptureBeforeImage { get; set; } = true;

    /// <summary>クリック操作後の撮影遅延（ミリ秒）。アプリ画面更新を待つ。</summary>
    public int PostClickDelayMs { get; set; } = 250;
}
```

### 5.6 機能要件

| ID | 要件 |
|---|---|
| FR-F301 | `ManualStep` の `ImagePath` を `AfterImagePath` にリネームし、`BeforeImagePath` を追加する |
| FR-F302 | `ProjectStep` の `ImagePath` を `AfterImagePath`、`ThumbPath` を `AfterThumbPath` にリネームし、`BeforeImagePath` / `BeforeThumbPath` を追加する |
| FR-F303 | `ProjectStore.ReadProjectJsonAsync` でv1.5.x の `imagePath` / `thumbPath` を自動移行する（FR-F3 の案 A または B を採用）|
| FR-F304 | `TriggerConfig` に `CaptureBeforeImage`（bool, 既定: true）と `PostClickDelayMs`（int, 既定: 250）を追加する |
| FR-F305 | `ManualStep.ImagePath` / `ProjectStep.ImagePath` / `ProjectStep.ThumbPath` を参照しているすべての箇所（`TriggerOrchestrator`, `ManualSessionRecorder`, `ExportService`, `HtmlManualWriter`, Writer 群, `ProjectViewWindow`）を新フィールド名に更新する |

## 6. FR-F4: ファイル保存仕様

### 6.1 before 画像のファイルパス

```
{project}/images/before/step_{StepNumber:D3}.png
{project}/thumbs/before/step_{StepNumber:D3}.jpg
```

フォルダが存在しない場合は自動作成する。

### 6.2 after 画像のファイルパス（変更なし）

```
{project}/images/step_{StepNumber:D3}.{ext}
{project}/thumbs/step_{StepNumber:D3}.jpg
```

既存の命名規則を維持し、`AfterImagePath` / `AfterThumbPath` の参照先として使用する。

### 6.3 before 画像の保存フォーマット

グローバル設定の `ImageFormat`（Png / Jpeg / WebP）に**関わらず**、before 画像は常に PNG として保存する。

`FileStorage.SaveAsync` または `CaptureService.Encode` の呼び出し側で `ImageFormat.Png` を明示的に渡すことで実現する。

### 6.4 before 画像のサムネイル

`ThumbnailService.GenerateAsync` を before 画像にも適用する。
サムネイルの最大幅は after と同じく `ProjectConfig.ThumbnailMaxWidth`（既定 320px）とする。

### 6.5 手動ステップ追加との統合

`ProjectViewWindow` の「＋ステップを追加」で before 画像ファイルも選択可能にする（オプション）。
選択された場合は `images/before/` にコピーして `BeforeImagePath` に設定する。

### 6.6 機能要件

| ID | 要件 |
|---|---|
| FR-F401 | before 画像を `{project}/images/before/` に PNG で保存し、`ProjectStep.BeforeImagePath` に相対パス（`images/before/step_NNN.png`）を設定する |
| FR-F402 | before サムネイルを `{project}/thumbs/before/` に生成し、`ProjectStep.BeforeThumbPath` に設定する |
| FR-F403 | `CaptureService` または `ManualSessionRecorder.RecordProjectStepAsync` で before 保存時に `ImageFormat.Png` を明示的に使用する |
| FR-F404 | `RecordProjectStepAsync` が before パスと after パスの両方を受け取れるよう引数を拡張する |
| FR-F405 | ステップ削除時（`BtnDeleteStep_Click`）に `BeforeImagePath` も `images/before/_deleted/` へ移動する |
| FR-F406 | `ProjectStore.CloneStep`, `MergeProjectsAsync`, `SplitProjectAsync` で `BeforeImagePath` / `BeforeThumbPath` のコピーも行う |

## 7. FR-F5: エクスポート対応

### 7.1 手順書（Markdown / Word）

各ステップで before → after の順に 2 枚の画像を出力する。

```
### ステップ N

**操作前**

![操作前](images/before/step_NNN.png)

**操作後**

![操作後](images/step_NNN.png)

{説明文}
```

before 画像が null（未取得）のステップはこれまでと同じく after 画像のみを出力する。

### 7.2 HTML エクスポート

各ステップに before / after の画像を横並び（または縦積み）で表示する。

```html
<section class="step">
  <h2>ステップ N</h2>
  <div class="images">
    <figure class="before">
      <figcaption>操作前</figcaption>
      <img src="data:image/png;base64,{base64_before}" alt="操作前">
    </figure>
    <figure class="after">
      <figcaption>操作後</figcaption>
      <img src="data:image/{mime};base64,{base64_after}" alt="操作後">
    </figure>
  </div>
  <p class="step-desc">{説明文}</p>
</section>
```

before 画像が null の場合は `<figure class="before">` を出力しない。

### 7.3 動画（APNG / MP4）

動画フレームには **after 画像のみ**を使用する（before 画像は使用しない）。
`VideoGenerator` / `FrameRenderer` は `ManualStep.AfterImagePath` を参照する。

### 7.4 画像エクスポート（ExportService.ExportImagesAsync）

before 画像も `exports/{ts}_images/before/` サブフォルダに含める。

```
exports/20260528_123456_images/
    001_step_001.png          ← after（アノテーション焼き込み済み）
    002_step_002.png
    ...
    before/
        001_step_001.png      ← before（アノテーションなし・PNG 元画像）
        002_step_002.png
        ...
```

### 7.5 アノテーションの適用範囲

| 画像種別 | アノテーション |
|---|---|
| after 画像 | 適用する（既存動作を維持） |
| before 画像 | **適用しない**（証跡として元画像のまま） |

`ExportService.BuildAnnotatedSession`, `ResolveAnnotatedImagePath`, `HtmlManualWriter` は
after 画像にのみ `AnnotationRenderer.Render` を呼び出す。
before 画像は常にファイルの直接参照またはそのままの Base64 変換を使用する。

### 7.6 機能要件

| ID | 要件 |
|---|---|
| FR-F501 | `MarkdownManualWriter` でステップごとに before → after の順で画像を出力する（before が null の場合は after のみ） |
| FR-F502 | `DocxManualWriter` で同様に before / after を出力する |
| FR-F503 | `HtmlManualWriter` で before / after を横並び表示する CSS を追加する |
| FR-F504 | `ExportService.ExportImagesAsync` で before 画像を `before/` サブフォルダにコピーする |
| FR-F505 | `VideoGenerator` / `FrameRenderer` は `AfterImagePath` のみ参照する（before は使用しない） |
| FR-F506 | `BuildAnnotatedSession` / `ResolveAnnotatedImagePath` は after 画像にのみアノテーションを適用する |

## 8. FR-F6: ProjectViewWindow 表示対応

### 8.1 ステップ詳細エリアの変更

before 画像が存在する場合、ステップ詳細エリアのアノテーション画像表示領域の上部に
before 画像をコンパクトに表示する。

```
[before 画像（縮小表示・読み取り専用）]
────────────── 操作前 ──────────────
[after 画像（既存アノテーション編集領域）]
────────────── 操作後 ──────────────
ツール: [番号] [矢印] [矩形] [吹き出し]
...
```

before 画像が null のステップは before エリアを非表示にする（`Visibility.Collapsed`）。

### 8.2 サムネイル一覧

`StepViewModel.ThumbImageSource` を `AfterThumbPath` に変更する。
before サムネイルはステップリストには表示しない（詳細エリアのみ）。

### 8.3 機能要件

| ID | 要件 |
|---|---|
| FR-F601 | ステップ詳細エリアに before 画像表示用 `Image` コントロールを追加する（読み取り専用、アノテーション操作不可） |
| FR-F602 | before 画像が null のステップでは before 表示エリアを非表示にする（`Visibility.Collapsed`） |
| FR-F603 | `StepViewModel` の `ThumbImageSource` を `AfterThumbPath` から読み込むよう更新する |
| FR-F604 | `ProjectViewWindow.LoadAnnotationImage` で before 画像と after 画像をそれぞれのコントロールにロードする |

### 9.1 設定ウィンドウ「撮影トリガー」タブへの追加

## 10. 非機能要件

| ID | 要件 | 実装方針 |
|---|---|---|
| NF-F01 | フックコールバックを遅延させない | before 撮影は必ず `Task.Run` で非同期実行。after 遅延は `System.Threading.Timer` で実装 |
| NF-F02 | before 画像は PNG 固定 | `CaptureService.Encode` 呼び出し時に `ImageFormat.Png` を明示指定。グローバル設定を参照しない |
| NF-F03 | before 画像にアノテーション・オーバーレイを付与しない | `FireBeforeCapture` でマスキング以外の加工を一切行わない |
| NF-F04 | before が取得できなくても after の記録を継続する | `_pendingBeforeShots` に before がない場合は `BeforeImagePath = null` で記録 |
| NF-F05 | 既存 project.json の移行 | `ProjectStore.ReadProjectJsonAsync` でロード時に `imagePath` → `afterImagePath` を自動変換 |
| NF-F06 | ストレージ増加への配慮 | before は PNG（無圧縮ではなく Deflate 圧縮 PNG）を使用。after が PNG の場合の増加は最小限。`CaptureBeforeImage = false` で無効化も可能 |
| NF-F07 | `_pendingBeforeShots` の孤立防止 | `before` 撮影から `PostClickDelayMs + 2000ms` 以上経過した未使用エントリは定期クリーンアップ（`TriggerOrchestrator.Dispose` またはタイマー） |
