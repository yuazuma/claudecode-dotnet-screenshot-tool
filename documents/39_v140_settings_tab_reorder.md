# v1.4.0 設定ウィンドウ タブ順序変更

## 変更ファイル

- `src/AutoScreenshot/Views/SettingsWindow.xaml`

## 変更内容

TabControl 内の TabItem ブロックの並び順を変更。コードビハインドの変更なし。

## 新旧対照

| 旧順 | 新順 | タブ名 |
|---|---|---|
| 1 | 1 | 一般（変更なし） |
| 2 | 2 | 撮影トリガー（変更なし） |
| 3 | 3 | 保存（変更なし） |
| 4 | 4 | プライバシー（変更なし） |
| 5 | **10** | **メタデータ**（後ろへ） |
| 6 | **9** | **通知**（後ろへ） |
| 7 | **5** | **手順書生成**（前へ） |
| 8 | **6** | **LLM連携**（前へ） |
| 9 | **7** | **動画生成**（前へ） |
| 10 | **8** | **プロジェクト**（前へ） |

## グループ構成

```
[一般] [撮影トリガー] [保存] [プライバシー]  ← グループB: キャプチャ設定
[手順書生成] [LLM連携] [動画生成] [プロジェクト]  ← グループC: 出力・生成
[通知] [メタデータ]  ← グループD: 補助・詳細
```

## 変更理由

- 「手順書生成」→「LLM連携」を隣接: LLM連携は手順書生成の品質設定であり同グループが自然
- 「プロジェクト」を出力グループ末尾に: ON/OFFで手順書生成・動画生成の動作が変わる上位設定
- 「通知」「メタデータ」を末尾に: 設定頻度が低い補助的タブ
