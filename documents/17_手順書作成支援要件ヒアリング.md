# 手順書作成支援 追加機能 要件ヒアリング

## 日時
2026-05-26

## プロンプト要約
「追加改修として、『操作手順書の自動生成』機能を組み込みたいと思います。自動撮影したスクリーンショット画像と、撮影した対象のウィンドウやボタンといったUIの名称、ラベル文字列、マウスカーソル座標、クリックイベントやキーイベントの種類、推測した操作目的といった情報を Markdown and/or docx 形式の操作手順書にまとめるものです。私にヒアリングして、./requirements/追加要件_手順書作成支援.md に追加機能の要件定義を書き出してください。」

---

## ヒアリング結果

### 生成タイミング
- **回答**: 両方対応（セッション終了時に自動 + メニューから手動）

### UI 要素情報の取得
- **回答**: UIA (Windows UI Automation) と OCR を組み合わせ
  - UIA が使えない場合は OCR にフォールバック
  - 両方失敗時は `<!-- TODO -->` レビューマークを埋め込む

### 操作目的の推測
- **回答**: 段階的対応（フェーズ1はルールベース、後で LLM オプション追加）

### 出力形式
- **回答**: Markdown (.md) と Word (.docx) の両方

### 章立て基準
- **回答**: アクティブウィンドウ切替で章区切り + タイムギャップ（既定 5 分）で時間小見出し

### スクリーンショットの含め方
- **回答**: 設定で切り替え可能（全ステップ / ウィンドウ切替時のみ / なし）

### 記載イベント（複数選択）
- **回答**: クリック（左・右・中）、ドラッグ&ドロップ、キーボード入力、ウィンドウ切替
  - 画面差分検知・手動撮影は記載対象外

### 保存場所
- **回答**: 設定画面で任意のパスを指定（既定: `{SaveFolder}/manuals/`）

### LLM 設定
- **回答**: 設定画面に API キー入力欄を追加

### 表紙情報（複数選択）
- **回答**: セッション日時、タイトル（手動入力）、OS/ユーザー名、ダイジェスト（LLM 生成）

### キーボード入力の記載方法
- **回答**: 両方対応（実文字列・キーコードを設定で切替）

### セッション区切り方
- **回答**: 両方（AutoScreenshot 起動時に自動開始 + 手動刺入も可）

### テンプレート
- **回答**: ユーザーテンプレート対応（Markdown テンプレート / Word .dotx）

### Word 書式レベル
- **回答**: 中程度（表紙・目次・ページ番号・ヘッダー/フッター）

### UI 取得失敗時の対応
- **回答**: できる限り記載 + 要レビューマーク（`<!-- TODO: UI名を確認してください -->`）
