# v1.6.0 操作前後スクリーンショット実装可否の検討

## ユーザープロンプト

```
クリックイベントやキー押下イベントを拾う際に、クリック前とクリック後のイベントを
分離して取得・画像撮影・保存できるようにすることは可能ですか？実装にはまだ反映させないでください。
```

---

## 調査対象ファイル

- `Services/HookService.cs`
- `Services/TriggerOrchestrator.cs`

---

## 現状のフロー（調査時点）

```
WM_LBUTTONDOWN → _lbDownTime 記録のみ
WM_LBUTTONUP   → MouseEvent → FireCapture → Task.Run → 撮影・保存
WM_RBUTTONDOWN → MouseEvent → FireCapture（即時）
```

**撮影は「クリック後（UP）」の1枚のみ。**

---

## 実現可能性の結論

**実現可能。** 既存アーキテクチャへのピンポイント拡張で対応できる。

---

## トリガー別 before/after タイミング

| トリガー | before | after |
|---|---|---|
| 左クリック | `WM_LBUTTONDOWN` | `WM_LBUTTONUP` + 遅延 N ms |
| 右クリック | `WM_RBUTTONDOWN` | `WM_RBUTTONUP` + 遅延 N ms |
| 中クリック | `WM_MBUTTONDOWN` | `WM_MBUTTONUP` + 遅延 N ms |
| ドラッグ | `WM_LBUTTONDOWN` | `WM_LBUTTONUP`（閾値超え）+ 遅延 N ms |
| キーボード | 新シーケンス開始（アイドルタイマー未稼働時の最初のキー） | `KeyboardIdleSeconds` 後（変更なし） |
| ウィンドウ切替 | 非対象 | — |
| 画面差分 | 非対象 | — |

---

## データモデル変更の必要性

| v1.5.x | v1.6.0 | 意味 |
|---|---|---|
| `ImagePath` | `AfterImagePath` | 操作後（メイン・アノテーション可） |
| `ThumbPath` | `AfterThumbPath` | 操作後サムネイル |
| —（新規） | `BeforeImagePath` | 操作前（PNG 固定・証跡） |
| —（新規） | `BeforeThumbPath` | 操作前サムネイル |

---

## 懸念点・トレードオフ

| 項目 | 内容 |
|---|---|
| after 遅延の適切値 | アプリごとに応答速度が異なる。設定可能にして対応 |
| ファイル数が倍増 | before は PNG（Deflate 圧縮）。`CaptureBeforeImage = false` で無効化可能 |
| step 記録の原子性 | before 撮影後クラッシュでも before 画像のみ孤立。project.json 書き込みは after 完了後 |
| 旧 project.json 互換 | `imagePath` フィールドを後方互換シムで `afterImagePath` に自動移行 |
| キーボード before | アイドルタイマーが未稼働の状態（`_inKeyboardSession == false`）を「入力開始」として検知 |
