# v1.6.0 操作前後スクリーンショット要件定義書作成

## ユーザープロンプト

```
以下の前提で、要件定義書を作成し、./requirements/追加要件_前後イベント.md に保存してください。
データモデルの変更については、最小限の変更というのではなく、命名規則・標準的な実装として
妥当な方法を採用してください。
before 画像は再利用性が重要なので、劣化しないようにPNG形式が必須です。before 画像には
アノテーションは付与せずに、証跡として元画像のままステップに追加してください。
```

## 要件書の構成

| セクション | 内容 |
|---|---|
| FR-F1 | 操作前スクリーンショット取得（タイミング・仕様・機能要件） |
| FR-F2 | 操作後スクリーンショット取得（遅延撮影・タイミング変更） |
| FR-F3 | データモデル変更（命名規則統一・後方互換移行） |
| FR-F4 | ファイル保存仕様（フォルダ構成・PNG 固定） |
| FR-F5 | エクスポート対応（MD / DOCX / HTML / 画像エクスポート） |
| FR-F6 | ProjectViewWindow 表示対応 |
| NF | フックコールバック非ブロック・PNG 固定・アノテーション非付与など |
| 実装ガイド | 変更対象クラス一覧 |

---

## 主要な前提・制約の反映

| 前提 | 要件への反映 |
|---|---|
| before は PNG 固定 | FR-F101, NF-F02: `ImageFormat.Png` を明示指定 |
| before にアノテーション付与しない | FR-F3, FR-F5: `AnnotationRenderer` は after のみに適用 |
| 命名規則を標準的に | FR-F3: `ImagePath`→`AfterImagePath`、`ThumbPath`→`AfterThumbPath` に統一 |
| 旧 project.json 互換 | FR-F303: ロード時に `imagePath`/`thumbPath` を自動移行 |
